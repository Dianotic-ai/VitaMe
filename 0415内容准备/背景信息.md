
### 产品名称：VitaMe 属于我的生命方案

  ### 产品核心定位：一个面向亚健康人群的健康状态监测与保健品智能顾问系统

  解决的核心问题：
  - 普通人看不懂体检报告的趋势变化，无法形成长期健康画像
  - 保健品市场信息不对称 — 同成分价差一倍、英文成分名难以检索、中文翻译不精确
  - 服用方式困惑（饭前/饭后、水溶/片剂）
  - 成分差异不透明（氧化镁 vs 甘氨酸镁等）
  - 缺乏针对个人病史的禁忌提醒（如结石患者不能补钙）
  - 来源合规性和价格合理性无从验证

  目标用户群：
  - 中老年人（体检报告积累多年，需要趋势分析）
  - 宝妈（关注家庭成员健康）
  - 健身人群（补剂选购）

  三大 Agent 模块：
  1. 保健建议 Agent — 体检报告分析 + 主观症状 → 综合健康评估
  2. 保健品选购/推荐 Agent — 成分解析、比价、合规性验证、个性化推荐
  3. 时间安排 + 打卡 + 情绪鼓励 Agent — 服用提醒、习惯养成、心理支持

  ### 产品愿景：让用户对自己的健康状态拥有掌控感

  1. 痛点分析
     (a) 常见病症：比如腰肌劳损、结节、各种身体疾病或视力问题等。
     (b) 个人感悟：就像我现在 30岁了，以前年轻时久坐几个小时都没感觉，但最近开始觉得腰痛，或者身体各处莫名疼痛，才发现以前的生活习惯很不健康。
     (c) 核心逻辑：如果身体第一时间发出痛感，我们就应该立马去调整。

  2. 功能设想
     (a) 提供健康生活习惯建议，比如每隔一个小时提醒用户站起来活动。
     (b) 监控饮水量，设定达标标准。
     (c) 针对不同年龄段和身体状况，推荐每日所需的微量元素及补充方式。

  3. 项目现状与计划
     (a) 文档缺失：目前关于调研文档、竞品分析和商业模式的内容全部都没有。
     (b) 核心目标：我们想借助 Harmes Agent 项目https://github.com/NousResearch/hermes-agent快速构建一个产品，参加 4月底由世界人工智能大会组委会举办的“超级个体创业黑客松”。
     赛事研究内容参考@research-hackathon-strategy.md
     (c) 技术路径：暂不确定，初期考虑做微信小程序，后续可能会开发 App希望接入智能手表等穿戴设备。


  我们项目的目标是做一个能为亚健康人群监测健康状态的系统。它包含体检报告分析、保健品推荐、成分分析以及饮食注意事项等功
  能，帮助用户关注自身状态，并在发现问题时逐步进行调整。

  举个例子，我的搭档上传了自己近十年的体检报告，综合分析出身体状况的问题。同时，他将最近的主观体验也全部描述了一遍，比
  如眼睛不舒服、肠胃疾病，以及生活和工作的压力等等。

  在此基础上，他又提供了几个他看过的选购保健品的视频，请 AI 帮他推荐：
  1. 解决比价与成分疑惑：他在某些网站比价时发现，同样的包装价格却能差出一倍。而且很多成分我们看不懂，必须用原始的英文词
  条去检索，翻译成中文后往往检索得不明确。
  2. 明确服用方式：比如保健品该饭前还是饭后吃？应该选择水溶性的、片装的，还是其他形式？
  3. 辨析成分差异：同样是补镁，为什么有的成分是氧化镁，有的是甘氨酸镁？这类成分差别为什么这么大？
  4. 针对性推荐：针对当前症状（如患有结石或肠胃疾病），哪些成分推荐，哪些不能推荐？

  这些问题都是普通人考虑不到的，医生推荐时通常也不会详细告知。同时，我也无法确认当前保健品的来源是否合规，或者医院是否
  溢价收取的资金。

  我希望能通过这个项目，让用户对自己的健康状态享有一种掌控感。
