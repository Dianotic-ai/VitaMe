## 二、今天的新发现：产品边界扩展

今天讨论中，我们提出了**比 Kevin 提案还大**的产品边界。Kevin 的提案仍以"保健品排雷"为核心（只是从表单变成对话）；而新方向是：

> 对话开场先问 **"最近感受如何？身体、心理有没有不舒服？"**
> → 有不舒服 → 追问症状、病史、体检报告
> → 输出**物理疗法（热敷眼睛）+ 饮食疗法（鱼肝油）+ 保健品 + 生活方式**的个性化方案

这把产品从"保健品安全排雷 Agent"**扩展为"融合型身心健康顾问 Agent"**。保健品排雷从主场景降为对话的一个分支。

## 三、锁定的 7 个决策

| # | 维度 | 决策 | 理由 |
|---|---|---|---|
| 1 | 用户路线 | **V1 自己 → V2 家人 → V3 社交** | 自己用爽了才有动力给父母推；Kevin 自己就是用户，Demo 更真实 |
| 2 | 产品形态 | **融合型 Agent 对话**（症状入口 + 产品入口 + 体检入口自然切换） | 一个对话界面搞定一切；用户说啥，Agent 自然切模式 |
| 3 | 合规边界 | **给剂量 + 强免责 UI**（如"鱼肝油 1000 IU/天"+ 醒目条"参考范围，建议咨询医生"） | 只给方向 = ChatGPT，给剂量才有实际价值；靠 UI 兜底合规 |
| 4 | 空场景应对 | **三合一兜底**：Agent 根据对话切换（预防建议 / 切工具模式问保健品 / 问体检报告） | 用户说"挺好的"也不浪费，永远有下一个价值点 |
| 5 | 症状追踪 | **MVP P0 必做**（不是 Sdu 原方案的 P3） | Kevin 主张：感知改善闭环 = 留存核心，没留存 = 漏桶 |
| 6 | MVP 范围 | **极致 MVP**：只做融合对话 + 禁忌检测 + 疗法推荐 + 免责 + 症状追踪。**不加家庭多成员、不加 OCR、不加中成药** | 2 周 Demo 时间紧，聚焦对话 wow moment |
| 7 | 技术栈 | **Next.js 单页 + 腾讯云硅谷服务器 + 海外域名** | 海外服务器无 ICP；微信可直接访问网页；Demo 分享链接即演；未来迁小程序核心逻辑可复用 |

## 四、MVP 功能清单（2 周，只有一个对话界面）

### P0（一定要有）

1. **融合对话 Agent**
   - 开场："最近感受如何？身体、心理有没有不舒服？"
   - 用户说症状 → 追问部位/频率/持续时间 → 问病史/在服药物 → 输出个性化方案（疗法 + 保健品 + 生活方式）
   - 用户说"我想查 XX 保健品" → 切工具模式，3 秒给禁忌检测结果
   - 用户说"还好" → 预防建议 / 问体检 / 问保健品（Agent 根据对话节奏三选一）

2. **禁忌检测规则引擎**：15 条硬编码高频禁忌 + SUPP.AI API（5.9 万药物-补充剂交互免费）+ DDInter（30 万药物交互，预算许可加）

3. **置信度分级**：红 / 黄 / 灰 / 绿四级（采纳 Sdu v1 方案）

4. **疗法推荐**：
   - 物理疗法：热敷、按摩、拉伸
   - 饮食疗法：食补 + 保健品
   - 生活方式：睡眠、运动

5. **剂量 + 强免责 UI**：所有剂量建议必须附"参考范围 X-Y，建议咨询医生"的醒目条，**不可折叠**

6. **症状追踪极简版**：
   - 用户对话中提到的症状自动入库
   - 每天 / 每周 Agent 主动追问"上次你说腰疼，现在怎么样？"
   - 展示简单趋势（文本或小曲线）

7. **免责声明**：每次 AI 输出底部强制展示（UI 层，非条款）

### 不做（V2 再加）

- ❌ 家庭多成员管理
- ❌ 体检报告 OCR
- ❌ 中成药成分库
- ❌ 社交裂变
- ❌ 电商导流 / CPS 佣金（**永不做**，参考 Sdu 的合规分析）
- ❌ 小程序版本

## 五、技术栈和部署

| 层 | 方案 |
|---|---|
| 前端 | Next.js 14（单页聊天界面，极简，不用 shadcn 全家桶） |
| 后端 | Next.js API Routes（或 FastAPI，看 Sdu 习惯），托管在腾讯云硅谷现有服务器 |
| 数据库 | SQLite（MVP 够用）或 Supabase（想省事就上） |
| LLM | DeepSeek-V3（成本低、中文好）+ prompt engineering |
| 数据源 | SUPP.AI API（免费）+ 15 条硬编码禁忌 + DeepSeek 作为 fallback（带置信度标灰） |
| 部署 | 腾讯云硅谷现有服务器 + 新买域名（.com 优先，几十块/年） |
| 访问路径 | 微信搜索域名 → "访问网页" → 内置 WebView 打开；或直接分享链接 |

**关键技术点**：微信搜索会识别任何 URL 格式，自动在结果顶部加"访问网页"入口，点击用内置 WebView 打开。**不需要任何微信官方资质或配置**。

## 六、Demo 叙事（90 秒）

**主线**：Kevin 的"14 轮对话买错 2 个"真实故事（主角即用户）
**辅线**：Sdu 的"银杏 + 阿司匹林"场景（展示禁忌恐惧钩子）

**Demo 流程**：
1. Kevin 现场说："我腰疼眼干心悸，跟 ChatGPT 聊了 14 轮才买对" → 痛点建立
2. 打开 VitaMe → 输入"最近腰疼眼干、心悸"
3. Agent 追问：有无肾结石？胃溃疡？高血压？在吃什么药？
4. Agent 输出：镁缺乏可能性高 + 推荐**甘氨酸镁 200mg 睡前** + **热敷眼睛 10 分钟** + 强免责条
5. 用户："我刚买了 Doctor's Best 镁片" → Agent 切工具模式 → "这款含氧化镁，你肠胃敏感会加重腹泻，建议换甘氨酸镁"
6. 展示症状追踪界面："三天后再来 check in"

## 九、验证清单（Demo 前必须跑通）

1. **技术验证**：浏览器打开域名 → 聊天框渲染 → 输入症状 → DeepSeek 返回结构化结果 → SUPP.AI 返回冲突数据 → 前端渲染安全卡片 + 免责 + 剂量条
2. **合规验证**：每次 AI 输出必有强免责；剂量必有"参考范围"兜底；"暂无数据"场景标灰
3. **体验验证**：Kevin 跑一遍自己的真实症状（腰疼眼干心悸），Agent 输出和 14 轮 ChatGPT 结果相比更精准、更有人感
4. **Demo 验证**：90 秒讲完 Kevin 故事 + 展示融合对话 + 切工具模式 + 症状追踪界面，评委能 get 到 wow moment
5. **跨端验证**：微信搜索域名 → "访问网页" → WebView 内正常对话（不卡、不闪退）

## 十、开放问题（方向已定，但后续可能要回头讨论）

1. **心理健康范畴**：开场问"心理不舒服"是否包含焦虑 / 失眠 / 压力？如果是，可能触碰心理咨询资质边界，需要明确范围
2. **用户获客路径**：症状驱动 App 的冷启动怎么做？微信私域？小红书内容？（V1 先不考虑，Demo 结束后再定）
3. **商业化**：MVP 阶段不做变现，但长期（V2/V3）路径是 B2B（企业健康管理）、保险导流、增值订阅，需要跟 Sdu 对齐长期方向
4. **数据隐私**：用户症状 + 病史 + 体检属于敏感个人信息，存储加密和授权流程需要专门设计
