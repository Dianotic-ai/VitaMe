## 一、一句话定位

### "你给爸妈买的保健品，真的安全吗？"

30秒检查家人正在吃的药物和保健品有没有冲突，然后推荐更安全、更划算的替代方案。

### 定位转变对照

| 维度 | 原定位（已放弃） | 新定位（已确认） |
|------|-----------------|-----------------|
| 一句话 | "提前3年发现你的健康拐点" | **"你给爸妈买的保健品，真的安全吗？"** |
| 核心功能 | 体检趋势分析 + 保健品推荐 + 打卡 | **药物-保健品冲突检测 + 安全替代推荐** |
| 目标用户 | 25-35岁亚健康白领（给自己） | **25-40岁成年子女（给父母选购和管理）** |
| 痛点强度 | "选不对保健品"（Nice to have） | **"给父母买的东西会不会出事"（Must have）** |
| 差异化 | 成分形式级推荐（三合一串联） | **药物×保健品×病史 三角交叉安全校验** |
| 使用频率 | 体检1次/年（低频） | **每次给父母买新保健品都用（中频）** |
| 信任锚点 | 引用学术文献 | **医学检测博士审核 + 硬编码规则 + 数据源透明** |
| 传播方式 | "你吃的这几样冲突吗？" | **"我帮我妈查了有冲突！你也帮你爸妈查查？"** |
| 情感驱动 | 个人健康焦虑 | **对父母的保护本能（孝道文化）** |
| 竞品对标 | 蚂蚁阿福、丁香医生（红海） | **几乎无针对"家庭保健品安全管理"的产品（蓝海）** |

## 二、为什么这样定位

### 2.1 安全避险为主线的三个理由

**理由一：差异化最锐利。** 蚂蚁阿福（3000万月活）、丁香医生、小红书博主都在做"帮你了解保健品"，但没有人在做"帮你检查正在吃的东西会不会出问题"。全球范围内也没有一个产品同时做到：品牌中立推荐 + 成分形式分析 + 禁忌交叉校验 + 个性化，放在一个消费者工具里。

**理由二：结合"给父母买"的场景，安全焦虑是最强烈的。** 给自己买错保健品，浪费点钱。给父母买的保健品和他们正在吃的药冲突——这个后果承受不起。"省钱"是Nice to have，"安全"在涉及父母时变成Must have。

**理由三：我们有医学检测博士和护士。** 这两个人的背景完美匹配安全方向——可以审核禁忌规则，Demo时可以展示"安全规则库经医学博士审核"。如果做比价方向，这两个人的专业背景就浪费了。

### 2.2 省钱买对做钩子的逻辑

用户发现冲突后，下一个问题一定是"那应该买什么替代？哪里买最便宜？"

**安全是入口，选购是延伸。** 这是一个自然的流程，不是两个割裂的功能。

核心价值公式：**安全检测（建立信任）→ 成分解析（展示专业度）→ 替代推荐+比价（创造经济价值）**

这个顺序不能倒——先做比价再做安全就变成了购物导流工具，跟什么值得买竞争；先做安全就是健康信息服务，差异化立住了。

## 三、目标用户

### 主要用户画像

**25-40岁成年子女**——给父母（50-70岁）选购和管理保健品的人。

典型场景：
- 小王（30岁程序员）过年回家，发现妈妈桌上摆了5瓶保健品，妈妈还在吃降压药和阿司匹林
- 小李（35岁白领）在京东给爸爸买了鱼油，不确定跟爸爸的降血脂药会不会冲突
- 小张（28岁宝妈）想给婆婆买钙片，但婆婆有结石病史，不知道能不能补钙

这个群体的共同特征：
- 有购买力，愿意为父母健康付费
- 有健康焦虑——怕买错、怕吃出问题
- 信息不对称——看不懂成分、不了解禁忌
- 父母不会自己查——需要子女帮忙管理

### 用户需求验证来源

- Sunny搭档的真实经历（上传10年体检报告、对比保健品）
- 多位朋友的共同经历（给父母买保健品时的焦虑）
- 市场数据：保健品消费中30-39岁群体占比34%（含为父母购买）
- 64%养生人群会选择购买保健品，但信息获取渠道鱼龙混杂

## 四、核心痛点

| 痛点 | 严重度 | 现有解决方案 | 为什么现有方案不够 |
|------|--------|-------------|-------------------|
| **安全盲区**：不知道父母吃的药和保健品会不会冲突 | 🔴 极高 | 百度搜索/问医生 | 搜索结果矛盾、医生不会主动讲保健品禁忌 |
| **信息黑洞**：看不懂成分表、不知道成分形式差异 | 🟡 高 | 小红书/抖音种草 | 内容质量参差不齐、骗子多、无个性化 |
| **价格不透明**：同款产品不同平台价差一倍 | 🟡 中 | 手动比价 | 耗时30分钟+、跨平台比较困难 |

**核心洞察：** 大多数人给父母买保健品时最大的焦虑不是"买贵了"，而是"买错了可能伤害他们"。安全焦虑 > 价格焦虑。

## 五、解决方案

### 5.1 产品核心功能

**P0 — Demo必须有（MVP核心）**

| 功能 | 说明 | 用户价值 |
|------|------|---------|
| **家庭保健品安全检测器** | 输入：家人正在吃的保健品+药物+病史 → 输出：红/黄/绿三级安全评估+证据来源 | 30秒发现潜在危险，避免伤害 |
| **AI健康对话** | 用户追问"为什么冲突？" → DeepSeek V3.2回答+免责声明 | 理解原因，建立信任 |
| **免责声明系统** | 每次输出UI层强制展示，不藏在角落 | 合规+信任态度 |

**P1 — Demo加分项（省钱钩子）**

| 功能 | 说明 | 用户价值 |
|------|------|---------|
| **安全替代推荐** | 发现冲突后推荐更安全的替代品+成分形式解析 | 不只告诉你"不能吃"，还告诉你"应该吃什么" |
| **多平台价格参考** | 手动整理Top 20常见产品的京东/iHerb参考价 | 省钱 |
| **家庭保健品档案** | 为爸爸/妈妈分别建档，记录正在吃什么 | 长期管理，提升留存 |

**P2 — 黑客松后迭代**

- 体检报告趋势分析（原核心功能，降级）
- 服用打卡和提醒
- 拍照识别保健品瓶身/成分表（OCR）
- 家庭组（管理多个家人）
- 月度家庭健康简报
- 跨境保健品成分表翻译

### 5.2 核心Demo场景

**用户输入：** 妈妈正在吃的7样东西——钙片、鱼油、维生素D、辅酶Q10、银杏叶片 + 降压药（氨氯地平）+ 阿司匹林。病史：高血压。

**30秒返回结果：**

🔴 **红色警告（1个）**
- 银杏叶片 × 阿司匹林 → 两者都有抗凝血作用，同服显著增加出血风险
- 建议：立即停用银杏叶片，或咨询医生调整
- 证据来源：NIH NCCIH

🟡 **黄色注意（2个）**
- 鱼油(高剂量) × 阿司匹林 → 鱼油也有轻微抗凝作用，叠加需注意剂量
- 钙片 × 降压药 → 建议间隔2小时服用

🟢 **绿色安全（3个）**
- 维生素D + 钙片 → 推荐组合
- 辅酶Q10 → 与当前用药无已知冲突
- 鱼油(低剂量) + 钙片 → 安全

**延伸推荐（省钱钩子）：** 银杏叶片需停用 → 如果目的是改善记忆，推荐磷脂酰丝氨酸(PS)作为更安全的替代 → 京东¥89 vs iHerb¥145，省39%。

## 六、信任体系

信任是这个产品的第一关卡。用户把家人的健康信息交给一个新产品，必须从四个维度建立信任：

| 信任层 | 短期（黑客松Demo） | 长期（产品上线后） |
|--------|-------------------|-------------------|
| **人的信任** | 展示"医学顾问：XXX，医学检测博士" | 扩大医药专业顾问团 |
| **数据的信任** | 每条规则附带来源（NIH/PubMed/SUPP.AI链接） | 建立规则更新日志，公开透明 |
| **系统的信任** | 硬编码规则不依赖LLM，现场演示安全兜底 | 引入更多确定性逻辑，减少AI幻觉风险 |
| **态度的信任** | 免责声明大方展示，不隐藏在条款里 | 用户可举报疑似错误规则，社区驱动质量改进 |

**关键资源：** 已有医学检测博士和医院护士作为医学顾问。这是整个项目最大的杠杆。

## 七、竞争格局

### 7.1 为什么这个缝隙存在

**蚂蚁阿福（3000万月活）主动不碰保健品推荐。** 原因：保健品推荐的合规风险+责任暴露。蚂蚁健康业务负责人公开表态："我们的商业模式在内部争议很大，我们还没有答案。"

**字节跳动小荷AI医生** 同样模式——全面的健康AI但零保健品功能。

**大厂不碰 = 创业公司的结构性机会。** 前提是你必须构建足够的安全基础设施（硬编码规则、医学审核、免责声明）来管理这个风险。

### 7.2 竞品对比

| 竞品 | 做了什么 | 没做什么 | 与我们的差异 |
|------|---------|---------|-------------|
| 蚂蚁阿福 | 体检解读、AI问诊、医生连接 | 保健品推荐、冲突检测 | 我们补了它主动回避的空白 |
| Key科益家 | 健康问卷+基因检测→保健品推荐 | 只推自家产品、无冲突检测 | 我们品牌中立+做安全校验 |
| LemonBox | AI个性化维生素订阅 | 只卖自有DTC产品、无交互检查 | 我们不卖产品，做中立推荐 |
| SUPP.AI | 59,096条补充剂-药物交互数据 | 仅研究工具，非消费者产品 | 我们把学术数据变成消费者工具 |
| 小红书/抖音博主 | 成分科普、种草推荐 | 无个性化、无禁忌校验、质量不可控 | 我们做个性化+安全校验 |

### 7.3 三个独特优势

1. **蚂蚁阿福不做的事我们做** —— 大厂主动回避的领域就是创业公司的机会
2. **全球无"药物×保健品×病史"三角交叉校验消费者工具** —— SUPP.AI是学术工具，Persona只推自有品牌
3. **有真实医学顾问** —— 医学检测博士+医院护士，Demo一行字的信任价值超过所有代码

## 八、商业模式

### 8.1 原CPS佣金模式已放弃

**放弃原因：**
- 实际佣金率仅2-5%（非假设的5-15%），每MAU月收入仅¥0.23
- 《广告法》第46条风险：利用健康知识变相发布保健品广告可罚100万+
- 丁香医生、薄荷健康、Keep无一家靠CPS成功

### 8.2 新商业模式

| 层级 | 模式 | 收入预期 |
|------|------|---------|
| **免费层（获客）** | 基础冲突检测免费，通过社交裂变获客 | ¥0（目的是用户增长） |
| **订阅层（留存）** | 深度报告+家庭档案+成分形式解析 ¥9.9-19.9/月 | 3%转化 × ¥14.9均价 = ¥0.45/MAU |
| **B2B层（规模化）** | 企业健康管理 ¥50-500/员工/年 | 高毛利、可预测 |
| **导流层（增值）** | 保险导流 ¥50-200/合格线索 | 远超CPS的单位经济 |

**黑客松策略：** Demo中不展示具体变现，口述"免费获客→订阅留存→B2B规模化"即可。

## 九、技术架构

### 9.1 推荐技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | Next.js 14+ 部署到Vercel | 即时部署，不需ICP，微信二维码可打开 |
| 后端 | FastAPI (Python) 部署到Railway | 匹配Kevin的Python背景 |
| 数据库 | SQLite(开发) → Supabase(生产) | 最小化设置 |
| LLM | DeepSeek V3.2 | 比Claude Sonnet便宜10-35倍 |
| 禁忌数据 | SUPP.AI API + 硬编码关键规则 | 59,096条交互，免费 |
| 开发工具 | Cursor AI + Claude Code | 配superpowers+BMAD quick-dev框架 |

### 9.2 为什么不做微信小程序（黑客松阶段）

ICP备案最快3-4周，健康类目更久，4月底上线不可能。**先做Web App（微信二维码打开体验一致），同时并行申请ICP备案，黑客松后迁移。**

### 9.3 三层禁忌检查引擎

| 层级 | 数据源 | 作用 |
|------|--------|------|
| **第1层：SUPP.AI API** | 59,096条交互，2200万篇论文 | 覆盖面广 |
| **第2层：硬编码关键规则** | ~50条最危险交互（医学博士审核） | 安全兜底，不依赖LLM |
| **第3层：LLM验证** | DeepSeek V3.2 + 结构化提示 | 补充未覆盖的交互 |

### 9.4 三层安全护栏

| 层级 | 机制 |
|------|------|
| **输入过滤（LLM前）** | 紧急关键词→"请拨打120"；诊断请求→注入免责声明 |
| **系统提示工程** | 硬编码："始终包含免责声明""永不诊断疾病""用⚠️标记交互" |
| **输出过滤（LLM后）** | 正则扫描"治疗""治愈""诊断"等危险词；强制免责声明 |
