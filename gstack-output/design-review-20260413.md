## Pass 2: Interaction State Coverage — 2/10

### 问题：零状态定义

PRD和设计文档没有定义任何交互状态。这意味着开发者会自己发明。

### Fix: 交互状态表

| 功能 | 加载中 | 空状态 | 错误 | 成功 | 部分 |
|------|--------|--------|------|------|------|
| **Tab1 首屏** | 骨架屏(shimmer) | "录入你的第一份体检，开始健康之旅" + [录入体检]按钮 | "网络异常，下拉刷新" | 显示健康摘要卡片 | 只有1次体检: 显示单次解读，提示"再录入1次即可看趋势" |
| **AI对话** | 消息气泡内闪烁点"..." (流式输出时逐字显示) | "你好！我是VitaMe，你的AI健康助手。试试问我：'我的尿酸偏高怎么办？'" | "AI暂时繁忙，以下为通用建议..." + 预缓存回复 | 完整回复 + 免责声明 + 证据来源标注 | 流式输出中途断开: 显示已收到部分 + "回复未完成，请重试" |
| **体检录入** | 提交中spinner | "选择你想录入的指标" + 常见指标列表 | 数值异常: "请确认数值是否正确（正常范围: X-Y）" | "录入成功！" toast + 自动跳转趋势 | — |
| **趋势图** | 图表骨架屏 | "需要至少2次体检数据才能看趋势" + [录入历史数据]按钮 | "图表加载失败" + 重试 | 趋势线 + 风险标注 | 只有2个点: 简单线段 + "数据越多趋势越准" |
| **冲突检测** | "检测中..." + 盾牌动画 | "输入你正在吃的保健品，VitaMe帮你检查是否冲突" | "检测服务异常" + 重试 | 红/黄/绿/灰结果列表 | 部分成分未识别: "以下成分未在数据库中: [X]，建议咨询药剂师" |
| **成分查询** | 搜索中spinner | "搜索你感兴趣的成分" + 热门成分标签 | "未找到该成分" + "试试别的名字？" | 成分详情卡片 | — |
| **严重异常** | — | — | — | 全屏红色警告: "检测到异常指标！请立即就医" + 120急救电话按钮 + 不可关闭(需确认"我已知悉") | — |

**关键设计决策：严重异常的UI**

当硬编码规则触发"请立即就医"时，这不是一个普通的toast。它应该是：
- 全屏遮罩层（红色/深色背景）
- 大号文字："检测到严重异常指标"
- 具体指标和数值
- 120急救电话一键拨打按钮
- 附近医院导航按钮（微信地图API）
- 必须点击"我已知悉"才能关闭
- 这条记录永久保存在审计日志中

**修正后评分: 7/10** — 核心状态覆盖，但还缺少微信授权流程、首次启动引导等。

**关键洞察：首次用户不应该从Tab1（AI对话）开始。**

Tab2（冲突检测）才是最低门槛的入口。不需要录入体检数据，不需要注册，30秒就能看到结果。

**建议：** 首次用户（无体检数据）打开小程序时，自动定位到Tab2并展示冲突检测入口。有体检数据的回访用户默认打开Tab1。

**修正后评分: 7/10** — 有了情绪弧线，还需细化回访用户的旅程。

## Pass 4: AI Slop Risk — 6/10

### 分类：APP UI（工具型，任务导向）

VitaMe是功能性健康工具，不是营销页面。适用App UI规则。

### AI Slop检查清单

| 模式 | 风险 | 评估 |
|------|------|------|
| 紫/蓝渐变背景 | 高 | ⚠️ 健康类App常用蓝绿色系，容易落入AI模板 |
| 3列特性网格 | 中 | 两个Tab不太可能出现，但成分百科列表要注意 |
| 圆形图标+说明文字 | 高 | 热门成分标签不要用这个模式 |
| 全部居中对齐 | 中 | 对话界面天然左右分布，但Tab2首屏注意 |
| 统一大圆角 | 中 | 微信设计规范已有圆角，保持一致 |
| 装饰性色块/波浪分割线 | 高 | 健康App常见。不要用。 |
| Emoji作为设计元素 | 高 | 红/黄/绿严重度用色块或图标，不用Emoji |
| 左边彩色边框卡片 | 中 | 冲突检测结果卡片不要用这个 |
| 通用hero文案 | 低 | 小程序没有hero区域 |
| 千篇一律的节奏 | 低 | 两个Tab结构不同 |

### 具体设计方向

**VitaMe应该感觉像：** 一个你信任的药剂师朋友，专业但不冷漠，有温度但不花哨。

**不要像：** 丁香医生（太医疗/严肃）、Keep（太运动/活力）、通用SaaS模板。

**色彩方向：**
- 主色：暖绿色系（信任+健康，但不是医院绿）
- 危险：红色（禁忌警告）
- 警告：琥珀/橙色（注意事项）
- 安全：翡翠绿（已知安全）
- 未覆盖：中性灰（未检测范围）
- 背景：浅暖灰（不是纯白，更温暖）

**字体：** 微信小程序默认使用系统字体。标题加粗区分层级即可。

**修正后评分: 7/10** — 有了方向，缺少具体token定义（需要DESIGN.md）。

## Pass 6: Responsive & Accessibility — 3/10

### 微信小程序的特殊约束

微信小程序不需要考虑桌面响应式（只在手机端运行），但需要考虑：

| 维度 | 现状 | 建议 |
|------|------|------|
| 屏幕尺寸适配 | 未提及 | 用rpx单位（750rpx = 设计稿宽度），自动适配 |
| 安全区域 | 未提及 | iPhone刘海屏/底部安全区域需要padding |
| 键盘遮挡 | 未提及 | 输入体检数据时，键盘弹起后输入框不能被遮挡 |
| 大字体模式 | 未提及 | 支持微信系统字体缩放（老年用户） |
| 色盲适配 | 未提及 | 红/绿不能是唯一区分方式，需要图标+文字辅助 |
| 触摸目标 | 未提及 | 最小44x44pt（微信规范要求） |
| VoiceOver/TalkBack | 未提及 | 微信小程序支持aria-*属性，关键按钮需要标注 |

### 色盲适配（对健康产品特别重要）

红/绿色盲是最常见的色觉异常（约8%男性）。VitaMe的红/黄/绿/灰严重度系统**不能仅靠颜色区分**。

```
修改方案:
  🔴 红色禁忌 → 红色背景 + ⛔ 图标 + "禁忌"文字标签
  🟡 黄色注意 → 琥珀背景 + ⚠️ 图标 + "注意"文字标签
  🟢 绿色安全 → 绿色背景 + ✅ 图标 + "安全"文字标签
  ⚪ 灰色未覆盖 → 灰色背景 + ❓ 图标 + "未检测"文字标签
```

**修正后评分: 6/10** — 核心无障碍已标注，但缺少详细的aria标注规范。
