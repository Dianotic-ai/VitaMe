二、最适合团队落地的方式
不要只做成“一个 prompt”，而要做成 三件套：

1. 审美宪法
这是最高层原则，解决“什么是对的”。

例如：

风格关键词必须是：淡雅、克制、有生命力、有呼吸感、手绘但不幼稚

不允许出现：科技蓝炫光、强渐变、赛博、过度3D、装饰堆叠、廉价插画感

首页气质优先级：

温柔可信

高级克制

有机生命力

当代感

转化效率

这个层是给团队统一判断标准的。

2. 执行模板
这是给设计师、产品经理、AI 工具直接用的。

例如每次做页面都固定输出：

品牌气质总结

配色 token

Hero 区结构

插画风格建议

CTA 风格建议

页面留白比例

禁忌项

prompt 模板

这个层是给“做事的人”直接复用的。

3. 评审清单
这是给 review 用的，解决“怎么判断合不合格”。

比如：

第一眼是否像“有生命感的高端品牌”，而不是“工具网站”？

页面是否有足够留白？

是否存在超过 3 个抢眼视觉中心？

插画是否保持单线条、手绘、轻盈？

颜色是否控制在主色 + 次色 + 点缀色体系内？

CTA 是否过于互联网促销化？

是否保留 Anthropic 式安静、理性、克制？

这个层会非常适合你迁移到团队。

三、我先直接给你一个可落地的 Skill 雏形
下面这份你可以直接作为团队内部 skill 的初版。

Skill：种子发芽风格首页设计规范
1. Skill Purpose
用于为品牌官网首页、产品介绍页、App 落地页生成统一的视觉方向与设计规范。
目标风格：Anthropic 式克制高级感 + 有机生命力 + 淡雅手绘植物感。

2. Core Metaphor
种子正在发芽

寓意：

脆弱中蕴含力量

辨别之后的生长

每个人身体里都有自己的答案

产品不是“强干预”，而是“陪伴生长”

这个隐喻必须贯穿视觉、文案、图形语言与交互节奏。

3. Style Keywords
必须同时满足以下关键词：

Organic

Minimal

Quiet

Gentle

Thoughtful

Airy

Human

Warm

Editorial

Hand-drawn

禁止滑向以下风格：

Cyberpunk

Over-decorated

SaaS dashboard heavy

Neon gradient

Overly commercial

Cute/cartoonish

Luxury fashion arrogance

Wellness cliché

4. Color System
用途	色值	名称	规则
主色	#2D5A3D	深林绿	用于标题点缀、按钮、芽叶、重点视觉
次色	#8B6B4A	种子棕	用于辅助图形、线条、局部卡片
点缀	#4A90B8	溪水蓝	少量用于提示、过滤、流动、状态
背景	#FAF7F2	羊脂白	大面积背景，形成温暖留白
深色	#1C1C1C	墨黑	正文、标题、深背景区
配色规则
页面 70% 应为浅背景和留白

主色与次色为骨架，蓝色只能做小面积点缀

不允许使用高饱和撞色

不允许炫目渐变作为主体

5. Typography
推荐方向：

标题：带一点 editorial 气质的 serif 或高质量 humanist sans

正文：简洁、克制、可读性强

字号层级拉开，但不能“吼叫”

规则：

标题像“思考”，不是“广告”

避免粗黑大字堆砌

行距宽松，留出呼吸感

每个 section 尽量只保留一个视觉主句

6. Illustration Language
参考融合：

Paul Klee：几何但有机，轻盈却有哲思

Saul Steinberg：极简线条、一笔成形

当代植物手绘：细线条、叶脉、种子、豆荚、发芽结构

插画规则：

以单线条、轻手绘、结构感为主

图形应当“像在生长”，不是“像在装饰”

可用元素：

种子

裂开的种皮

细芽

豆荚

根系

叶脉

水流线

禁止：

拟人卡通

复杂写实植物堆叠

大片纹理噪点

过强装饰边框

7. Layout Principles
页面结构优先采用：

大留白

卡片式分区

非拥挤模块节奏

图文之间有“停顿感”

Hero section 建议结构：

左侧：核心标题 + 一句解释 + CTA

右侧：线条植物插图 / 种子发芽图形

下方：3 个价值点卡片

再往下：流程或状态变化（种子→发芽→开花→结果）

布局原则：

一屏只讲一件事

元素不要多，但每个元素都有意义

不追求信息密度，追求情绪和秩序

8. Component Language
Button
圆角适中

背景可用深林绿

hover 微弱变化即可

不要互联网促销风

Card
浅底

细边框或弱阴影

间距舒服

卡片像纸张，不像 dashboard 模块

Icon System
建议四个核心状态：

种子

发芽

开花

结果

可对应：

onboarding

insight

growth

outcome

9. Copy Tone
文案语气：

平静

清晰

不喊口号

有洞察

像一个成熟、有耐心的人在陪伴用户

避免：

“革命性”“颠覆式”“效率暴涨”

过度营销化表达

夸张价值承诺

更适合的表达：

帮助看见

陪伴生长

温和而坚定

从混乱中辨别方向

找到身体里的答案

10. AI Prompt Template
用于让 AI 生成首页视觉稿：

Prompt 模板：

Design a website homepage in an Anthropic-inspired editorial minimal style with organic hand-drawn botanical elements.
Core metaphor: a seed beginning to sprout, symbolizing discernment, inner growth, quiet strength, and life emerging from fragility.
Use a soft warm background (#FAF7F2), deep forest green (#2D5A3D), seed brown (#8B6B4A), and small accents of water blue (#4A90B8).
Visual language should feel calm, intelligent, airy, and human-centered.
Include generous whitespace, elegant typography, simple card-based layout, and fine line botanical illustrations inspired by Paul Klee, Saul Steinberg, and contemporary minimalist plant sketches.
Avoid flashy SaaS aesthetics, neon gradients, heavy 3D graphics, overly commercial UI, or clutter.
The page should feel premium, thoughtful, and quietly alive.

12. Review Checklist
评审时必须逐项检查：

是否一眼看上去“安静、高级、有生命力”？

是否保留了足够留白？

是否有明显的“种子发芽”隐喻？

插画是否轻盈克制，而不是装饰性堆砌？

色彩是否淡雅统一，没有跑偏？

标题是否像“洞察”，而不是“广告语”？

CTA 是否克制可信？

页面是否像 Anthropic 式品牌，而不是普通 SaaS 官网？

是否兼具人文感与现代感？

是否适合长期作为品牌标准扩展？

四、怎么迁移到团队最有效
我建议你在团队里不是“发一个 prompt”，而是做成下面 5 个资产：

1. 一个总 Skill 文档
类似我上面这份，作为统一规范。

2. 一个 Prompt 库
按场景拆开：

官网首页 prompt

App landing page prompt

品牌插画 prompt

Hero 区 prompt

Logo 草图 prompt

UI review prompt

3. 一个 Figma token 系统
把颜色、圆角、间距、阴影、字体层级都 token 化。
这样团队不是“凭感觉模仿”，而是直接套标准。

4. 一个评审 rubric
每次设计评审都按同一套 10 分项打分。
这样风格不会越做越散。

5. 一个示例库
保留：

合格案例

不合格案例

跑偏案例

修改前后对比

团队真正学会风格，靠的是示例，不只是描述。

五、你这个 skill 继续升级的话，最值得加的模块
接下来可以继续扩成一个完整的 品牌设计操作系统：

模块 A：品牌视觉圣经
定义品牌世界观、隐喻、色彩、版式、插画语法。

模块 B：首页生成器
输入业务目标，自动产出首页信息架构 + 视觉方向。

模块 C：设计评审官
任何设计稿进来，自动判断：

是否符合品牌

哪些地方太“互联网”

哪些地方太“花”

哪些地方缺少呼吸感

模块 D：多工具迁移层
同一套 skill 输出到：

ChatGPT / Claude 做 prompt

Figma 做设计规范

Lovable / Framer / Webflow 做落地页

前端团队做 design token
