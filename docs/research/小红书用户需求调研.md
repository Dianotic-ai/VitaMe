# 小红书用户需求调研

> 日期：2026-04-18
> 状态：Draft
> 调研来源：小红书补剂相关笔记真实用户评论（4 张截图 / 约 20 条代表性评论 / 107 条评论密度观察）
> 目的：为 VitaMe P0 定位提供真实用户声音验证，输出可直接用于 Demo 和规则引擎测试的种子数据

---

## 1. 调研背景

在 2026-04-17 完成 VitaMe P0 定位收敛（从"融合型身心健康 Agent"收敛到"补剂安全翻译 Agent"）之后，必须回答一个关键问题：

**真实用户是否有这些需求？需求密度如何？愿意为"安全翻译"这一层付出注意力吗？**

为此，选取小红书补剂科普赛道的一个头部博主笔记作为抓手，通过评论区观察真实用户的自然提问。

---

## 2. 调研对象

### 主样本

- 博主：**FoodieV**
- 笔记主题：补剂推荐/搭配（鱼油、维 D、D3、镁、益生菌、维 B、维 C 等多补剂共吃）
- 笔记量级：**6584 赞 / 7713 收藏 / 107 评论**
- 作者定位：海外食品/营养 KOL，有测评栏目

### 辅样本

- 博主：**momo（多囊减肥版）**
- 笔记主题：鱼油测评（#epa #omega3 #深海鱼油）
- 笔记量级：19 赞 / 10 收藏 / 3 评论
- 用途：用于病史精准场景补充验证（apoe4 基因 / 肝炎脂肪肝）

### 调研方法

- 4 张评论区截图（约 20 条代表性评论）
- 观察用户**自然提问的语言、情绪、问题结构**
- 对比 VitaMe P0 定位文档中推导出的问题类型，评估命中率
- **不是定量调研**，是密度+质量的定性判断

## 3. 十类真实问题及原始证据

每一类下附至少 1 条原始评论引用，按命中 P0 能力清单分类。

### 3.1 药物 × 补剂冲突

> "在吃 Eva 的话，也可以吃这么多药吗，会不会肝脏代谢不了嘞"
> —— 湖南，2025-08-07

**情绪特征**：担忧 + 代谢焦虑
**对应 P0 能力**：safety-judgment（药物交互）+ safety-translation（代谢路径翻译）

---

### 3.2 多补剂组合 × 时间安排 × 冲突

> "如果鱼油、维 D、D3、镁、益生菌、维 B、维 C，这些都吃，应该分别在每天哪个时间段吃比较好？或者相互需要间隔多久？我怕有冲突🥺"
> —— 贵州，2025-08-21

**情绪特征**：明确写出"我怕有冲突"，3 人点赞
**作者回复**："不太需要特别间隔，一起吃也没啥大问题。把水溶性（维 B、维 C、益生菌）放早上，脂溶性（D3、鱼油）跟餐吃，镁放晚上即可"（15 赞）
**对应 P0 能力**：safety-judgment（补剂互作）+ safety-translation（时间分配逻辑）

**KOL 回复的局限**：
- 不结合个人病史
- 不给"为什么"解释
- 下次换一种补剂就套不上

### 3.3 病史 × 补剂冲突（精准场景）

> "问问家里老人肝有问题可以吃鱼油吗？apoe4 可以吃什么样的鱼油？"
> —— 湖南，01-04

**情绪特征**：代父母决策 + 基因精准 + 老人慢病（三重恐惧）
**作者追问回复**："肝具体是什么问题呢，凝血功能有没有问题"
—— **这恰好是 VitaMe"关键风险采集"应该做的事**
**对应 P0 能力**：safety-judgment（病史交叉）+ archive-recheck（家人档案）+ User Journey J3

### 3.4 长期服用安全性

> "这些都可以一直吃吗？" —— 湖北，03-22
> "omega-3 可以长期吃吗？"（第 4 图系统"猜你想搜"高频联想词）

**对应 P0 能力**：P0 暂不做停药周期，但反映用户对"开吃后如何停"的焦虑，P1 可做"周期提醒 / 复评建议"

### 3.5 食物/饮品 × 吸收影响

> "早餐时一定要喝咖啡的，会影响随餐吃的补剂吸收吗？"
> —— 上海，2025-09-07

**对应 P0 能力**：safety-translation（吸收/代谢翻译）

### 3.6 家人场景

同 3.3 — "家里老人肝+apoe4+鱼油"。**一条评论同时命中 PRD §7 场景 C（替父母买）+ User Journey J3（家人场景）+ CLAUDE.md 原始定位"你给爸妈买的保健品真的安全吗"。**

### 3.7 成分形式差异

截图中没有直接问"氧化镁 vs 甘氨酸镁"，但作者回复"水溶性放早上 / 脂溶性跟餐"本身就在做**成分形式翻译**——用户需要这层翻译，KOL 只能给一句话结论不讲道理。**这正是 VitaMe 差异化能补齐的空白。**

### 3.8 检测前置（是否要先去医院）

> "请问补充这些补剂要先去医院检测是否缺少这些元素吗？" —— 上海，2025-09-08
> 社区自助回复："最好查个血常规" —— 四川，2025-09-10

**对应 P0 能力**：P0 不做（避免扩边界回到融合 Agent 老路），但记入 P1 候选

### 3.9 特定人群需求

> "内分泌啥的分享下嘞" —— 浙江，2025-11-10
> "女性补剂好像没说" —— 浙江，2025-11-10
> "鋅可以補充嗎？" —— 加拿大，2025-09-15（繁体，海外华人）

**对应 P0 能力**：P0 不做分群定制，P1 做女性/老人画像

### 3.10 品牌选择辅助

> "能推荐一些牌子吗" —— 河南，2025-11-13
> "请问鱼油和益生菌有推荐吗🥺" —— 广东，2025-07-16

**作者回复**："我主页有鱼油和维 D 的测评，搜索应该就能看到！"
**对应 P0 能力**：P0 明确不做 CPS 导购，但可用"成分形式推荐"间接满足（优先甘氨酸镁、避开氧化镁）

### 3.11 信息过载 / 决策瘫痪（bonus）

> "感觉全都要吃🥲" —— 广东
> "这些问题我都有🥲吃啥呀" —— 广东，01-08

**反映**：用户**无决策起点**的痛点，首页需要明确决策引导

## 4. 五个硬核洞察

### 洞察 1：VitaMe P0 定位 × 真实需求 = 100% 命中

0417 定位文档中推导出的 6 类问题（成分形式 / 水溶性脂溶性 / 体质 / 风险来源 / 替代方向 / 商家话术辨别），**每一类在真实评论中都能找到直接对应**。从假设走到事实。

### 洞察 2：KOL 模式的天花板 = VitaMe 的机会底板

FoodieV 给 angie 的回复获 15 赞："一起吃没啥大问题，水溶性早上、脂溶性跟餐、镁晚上。"

| KOL 回复的局限 | VitaMe 能补齐的 |
|---|---|
| 粗放（"没啥大问题"不结合个体） | 个人化（病史/用药/体质三角检查） |
| 107 条评论只能答十几条 | 即时（AI 可以每一条都答） |
| 纯文本回答，用户看完就忘 | 结构化（时间表 / 等级卡片） |
| 不解释为什么，下次换补剂就套不上 | 原因可追溯（每个建议有"为什么"） |
| 不考虑病史和用药 | 强制关键风险采集 |

### 洞察 3:"家人场景"真实存在且情绪重过自己用

第 4 张"家里老人肝有问题可以吃鱼油吗？apoe4 可以吃什么样的鱼油？"**不是好奇，是恐惧**。涉及：
- 老人（决策责任不可逆）
- 肝病（具体病史）
- apoe4（基因精准性）

三份文档（CLAUDE.md、PRD、User Journey J3）里的用户画像**被一条真实评论统一验证**。

### 洞察 4:"品牌推荐"需求绕不过但可以转化

多条评论问"能推荐一些牌子吗"。VitaMe P0 不做 CPS（合规+信任红线），但需求真实存在。

**解法（P1 候选）**：不推荐品牌，推荐成分形式（"优先看标签是甘氨酸镁，避开氧化镁"）—— 间接把用户导向"懂成分、不被品牌忽悠"的决策能力。

### 洞察 5:"检测前置"暗示决策链比产品边界更早

Qian 问"补充这些补剂要先去医院检测是否缺乏吗"—— 真实决策流程是：

**"感觉缺"→"纠结要不要先检测"→"纠结买什么"→"纠结怎么吃"**

VitaMe 当前只切入后两步。**暂不扩边界（避免回到融合 Agent 老路）**，但记入 P1 入口优化：首页可加"不确定要不要补？先查查这个补剂"的辅助引导。

## 5. 对 P0 定位的验证结论

### 5.1 强验证（做这个 P0 是对的）

| P0 定位维度 | 真实评论是否证实 | 证据 |
|---|---|---|
| 尖锐场景（买前判断） | ✅ | angie 多补剂怕冲突；爱琳父母肝病怕鱼油 |
| 合规边界（不诊断，做翻译） | ✅ | 用户要的是"能不能吃/怎么吃"，不是诊断 |
| 差异化（KOL 和 ChatGPT 都不做） | ✅ | KOL 粗放普适；ChatGPT 不结合病史+不沉淀 |
| 家人场景（子女决策焦虑） | ✅ | 第 4 图老人肝病+apoe4+鱼油 |
| 结果沉淀（档案+复查） | ⚠️ 未直接验证 | 评论层看不出"想沉淀档案"的表达 |

### 5.2 弱验证（可能存在误判）

| 产品假设 | 真实评论呈现 | 建议 |
|---|---|---|
| "用户愿意保存档案用于复查" | 评论里没看到这种表达 | 需要 3-5 个深度访谈验证 |
| "用户愿意回答 2-4 个病史问题" | 评论里倾诉式描述病史很常见 | 间接支持 |
| "用户要红黄灰绿等级" | 用户习惯的心智是"能 / 不能" | 色块可保留，文案避免过度专业 |

### 5.3 未验证的核心问题（必须通过访谈补）

1. **付费意愿**：用户为"AI 安全翻译"愿意付 0 / 9 / 29 元？
2. **切换成本**：用户愿意从小红书/ChatGPT 切到 VitaMe 吗？第一次门槛是什么？
3. **复查频次**：新增补剂时用户会想到回 VitaMe 吗？（留存飞轮核心假设）
4. **档案录入门槛**：病史+用药录入阻力有多大？

## 6. 对 MVP 执行方案的直接影响

### 6.1 Demo 叙事：重写方向

- **弃用**：Kevin "14 轮对话买错 2 个"（症状入口，和新定位错位）
- **采用**：真实小红书用户问题（陌生人场景 > 创始人场景）

Demo 开场候选：

> "小红书上一条鱼油测评有 6000+ 赞，评论区 107 条，密密麻麻都是这种问题：家里老人肝炎脂肪肝能不能吃鱼油？apoe4 要选什么样的鱼油？答好这些问题需要 3 秒，但 KOL 挤几天也答不完 107 条。VitaMe 正是为此而生。"

### 6.2 规则引擎测试用例

第 3.1 - 3.7 节的每一类真实问题都能直接变 spec 的 `#### Scenario: WHEN/THEN`。详见同目录《Demo 种子问题清单-20 条.md》。

### 6.3 用户画像需要重新对齐

0417 定位文档中"两类 Primary User（给自己 + 给父母）"**在真实评论里是混合而非分层的**：angie 自己吃多补剂；爱琳问家里老人；其他评论两种混杂。建议：

- **不强行分"V1 自己 / V2 家人"**，P0 支持两种场景共存
- 对话中用一个简单切换即可（"给谁查？"）
- 推翻 2026-04-16 v2 纪要里的"V1 先自己→V2 家人"顺序论

### 6.4 P1 候选列表（P0 不做但记下）

- 检测前置引导（首页"不确定要不要补？"）
- 长期服用 / 停药周期提醒
- 成分形式推荐替代品牌推荐
- 女性 / 老年人特定画像
- 孕期 / 哺乳期特殊群体

## 8. 一句话结论

**小红书 107 条评论的密度验证了 VitaMe P0 定位的真实性和尖锐度；KOL 模式的天花板即 VitaMe 机会的底板；下一步需 3-5 个深度访谈验证"档案沉淀 + 复查回访"这一核心留存假设。**
