# 用户旅程与 UI 设计

## 1. 信息架构

P0 只有一个主页面：

```text
/chat
  Header
  Optional RoutineSummaryStrip
  MessageList or EmptyState
  Contextual inline cards
  ChatInput
  DetailDrawer
  RoutineDrawer
```

独立页面 `/profile`、`/memory`、`/reminders` 不作为 P0 主路径入口展示。MVP 不提供多人档案 UI。

首访禁止元素：

- Reminder 中心入口。
- 4 格药盒装饰。
- 大段功能说明。

CTA 必须是 assistant 消息内联，不做底部功能 dock。

## 4. RoutineConfirmCard

当 assistant 消息出现早 / 中 / 晚 / 睡前结构时，在该消息下方展示确认卡：

```text
┌────────────────────────────────────────────┐
│ 这是我整理的今天怎么吃，你看一下           │
├────────────────────────────────────────────┤
│ 早    [B族：随早餐                  ✎]    │
│ 中    [叶黄素：随午餐，脂溶性        ✎]    │
│ 晚    [—                            ✎]    │
│ 睡前  [镁：睡前 1 小时               ✎]    │
├────────────────────────────────────────────┤
│        先放着        保存到我的提醒        │
└────────────────────────────────────────────┘
```

行为要求：

- 保存前不写 routineStore。
- 保存前不写 action memory。
- 保存前不创建 profile。
- 用户可以编辑每个 slot。
- 空 slot 不保存。
- 保存后卡片折叠成“已保存到提醒”。
- 保存后顶部 RoutineSummaryStrip 出现。

## 5. RoutineSummaryStrip

P1 后显示：

```text
┌────────────────────────────────────────────┐
│ 早 1 │ 中 1 │ 晚 - │ 睡前 1      今天药盒 › │
└────────────────────────────────────────────┘
```

规则：

- 只在 `routine.items.length > 0` 渲染。
- 空 slot 显示 `-`。
- 点击打开 RoutineDrawer。
- 不作为首访装饰。

## 6. Product Inspect Inline

用户粘贴 URL：

```text
输入框上方:
[ 正在抓取 iherb.com/... ]
```

成功后自动发送规格事实消息：

```text
我贴了一个商品页，请基于以下官网解析结果帮我判断：

商品名：...
品牌：...
每份剂量：...
成分：...
建议服用：...
警示语：...
剂型/工艺：...
缺失字段：...
```

这条消息是本轮临时上下文，不自动写 action memory。只有用户保存由该商品生成的 routine，或显式保存商品判断时，才写入 action memory。

失败时：

```text
抓不到完整页面。把官网详情页文字、Supplement Facts、营养成分表或包装背面说明粘进来，我继续帮你解析。
```

失败提示不能替代抓取尝试。P0 不做拍照入口；如果用户已经有包装文字，可以手动粘贴。

## 7. DetailDrawer

DetailDrawer 用于透明度，不是主路径：

- 本轮依据。
- 官网解析结果。
- 本地 action memory。
- 清空本地数据。

清空规则：

- “清空今天药盒”只清空 routine，让主路径回到 P0。
- “清空全部本地数据”清空 routine、action memory、商品解析缓存和对话。

禁止在 DetailDrawer 展示多人档案表单。即使测试覆盖家人场景，也只能作为对话上下文处理。

## 8. 视觉约束

- 手机优先。
- 不做 landing page。
- 不用大面积装饰卡片堆功能。
- 不用单一紫蓝或深蓝渐变主题。
- 文字必须在移动端不溢出。
- CTA 用上下文内联 chip 或确认卡，不做固定功能 dock。
