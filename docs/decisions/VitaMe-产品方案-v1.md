# VitaMe 产品方案

> 版本：v1.0 | 日期：2026-04-16
> 基于：9个AI Agent并行分析 + 4轮找茬评审 + 27个开源数据库调研 + 竞品全景扫描

---

## 一、产品定位

### 一句话

> **"你全家吃的药、保健品、体检结果，第一次放在一起看。"**

### 核心问题

医生只管开药，药店只管卖货，你只管下单——但**没有人把这些放在一起检查过**。

一个中国中老年人平均同时服用 2-3 种处方药 + 1-2 种中成药 + 2-4 种保健品/传统补品，来自至少 4 个互不知情的来源（医院、药店、电商、朋友推荐）。没有任何一个人或系统掌握全貌。

### 产品本质

**家庭健康全景系统** —— 第一次把一个人吃的所有东西（处方药 + 中成药 + 保健品 + 传统补品）和身体状况（慢性病 + 体检异常项）放在一个地方，持续交叉监控。

### 类比

- ChatGPT = 你问路人"这两个药能一起吃吗"，路人回答了，然后各走各路
- VitaMe = 你家药柜上贴了一张智能标签，每次放进新东西自动检查所有组合

### 为什么不是ChatGPT

| ChatGPT | VitaMe |
|---------|--------|
| 问完就忘 | 记住全家人的药柜 |
| 每次重新输入全部 | 新增一项自动交叉检查已有全部 |
| 不知道你的身体状况 | 结合体检报告+慢性病做个性化检查 |
| 不会说"我不知道" | 灰色标注"暂无研究数据" |
| 无法验证来源 | 每条结论标注数据库来源+置信度 |
| 生成文字段落 | 结构化安全卡片，可分享 |

## 二、目标用户

### 主要用户

**25-45 岁成年子女** —— 给 50-70 岁父母管理药品和保健品的人。

典型场景：
- 在小红书看到推荐，想给妈妈买鱼油，不确定跟降压药冲不冲突
- 发现爸爸药店买了三七粉，担心跟他的处方药有交互
- 妈妈换了一种降压药，需要重新检查所有保健品是否安全
- 春节回家发现父母药柜里一堆瓶瓶罐罐，想一次性查清楚

### 次要用户

- **35-55 岁自用保健品人群** —— 自己吃了多种保健品+有慢性病，想审计是否合理
- **管理家庭健康的宝妈** —— 同时关注孩子、丈夫、父母的健康

### 用户不是谁

- 不是健身人群（他们的需求是"怎么练"，Keep 在做）
- 不是减肥人群（他们的需求是"怎么吃"，薄荷健康在做）
- 不是需要医疗诊断的患者（去医院）

## 三、竞争格局

### 为什么是蓝海

**"中国处方药 × 西式保健品 × 中成药/传统补品 × 个人病史"四维交叉安全检查 = 全球零竞品。**

| 竞品 | 能做什么 | 不能做什么 |
|------|---------|-----------|
| Drugs.com | 英文药物交互检查 | 无中国药品、无中成药、无个性化 |
| ChatGPT/豆包 | 单次分析，中文回答 | 不记忆、不持久、不可验证、不会说"不知道" |
| 蚂蚁阿福 | AI 健康问答，3000万月活 | **主动不碰保健品推荐**（监管风险） |
| 丁香医生 | 在线问诊+科普 | 不做保健品，推荐=带货损害专业形象 |
| LemonBox | AI 个性化维生素订阅 | 只推自有品牌，非中立，无安全检查 |
| Keep AI 营养师 | AI 饮食+营养管理 | 不做保健品安全检查，聚焦运动人群 |
| 用药助手 | 专业级药物查询 | 面向医护，非消费者，保健品覆盖弱 |

### 大厂为什么不做

- **蚂蚁/字节**：主动回避保健品（监管风险+责任暴露）
- **京东/阿里**：电商基因，只想卖货，不会做"别买了"的反向建议
- **丁香医生**：推荐保健品=带货，损害专业形象
- **LemonBox/Persona**：品牌锁定，只推自有产品，无法中立

**VitaMe 不卖货、不推荐品牌 —— 中立性就是护城河。**

## 六、数据引擎

### 不依赖医学博士，依赖全球公开数据库

**核心引擎层：**

| 数据库 | 数据量 | 用途 | 免费 |
|--------|--------|------|------|
| DDInter 2.0 | 30万条药物交互 + 857条药食交互 | 药物-药物交互检查 | 是 |
| SUPP.AI | 5.9万条药物-补充剂交互 | 药物-保健品交互检查 | 是 |
| RxNav API | NLM免费API | 药物标准化识别+基础交互 | 是 |
| OpenFDA FAERS/CAERS | 数千万份不良反应报告 | 安全信号验证 | 是 |
| DSLD | 20万+补充剂标签 | 保健品成分查询 | 是 |
| PubChem | 1.11亿化合物 | 化合物统一ID和结构匹配 | 是 |

**中药/中成药层：**

| 数据库 | 数据量 | 用途 | 免费 |
|--------|--------|------|------|
| HERB | 7263中药, 49258成分 | 中药成分查询 | 是 |
| TCMSP | 499中药, 29384成分 | 中药成分-靶点映射 | 是 |
| NMPA数据查询 | 全部批准药品 | 中国药品名→通用名映射 | 是 |
| 蓝帽子平台 | 6.8万注册保健食品 | 中国保健食品合规验证 | 是 |

**循证补充层：**

| 数据库 | 用途 | 免费 |
|--------|------|------|
| PubMed API | 按需检索最新交互研究文献 | 是 |
| ClinicalTrials.gov | 查询相关临床试验 | 是 |

### 置信度计算逻辑

```
★★★★★  3+个独立数据库交叉验证       → 🔴 明确警告
★★★☆☆  1-2个数据库有记录            → 🟡 注意提示
★☆☆☆☆  仅文献提及/AI推断            → 🟠 有限证据，建议咨询医生
无数据   所有数据库均无记录            → ⚪ 暂无数据，建议咨询医生
无冲突   数据库明确标注无已知交互       → 🟢 未发现已知风险
```

### 中英药品映射方案

```
用户输入"硝苯地平缓释片"
  → NMPA数据库查通用名: 硝苯地平 (Nifedipine)
  → RxNorm/DrugBank匹配: Nifedipine (DB01115)
  → 用DrugBank ID查DDInter/SUPP.AI交互
  → 返回中文结果

用户输入"复方丹参滴丸"
  → 中成药数据库拆解: 丹参 + 三七 + 冰片
  → HERB/TCMSP查各成分靶点
  → 分别与其他药物做交互检查
  → 合并返回
```

### 安全架构

```
用户输入
  ↓
第一层：硬编码规则引擎（零LLM依赖，不会幻觉）
  ↓ 命中 → 直接返回结果
  ↓ 未命中
第二层：多数据库交叉查询（SUPP.AI + DDInter + OpenFDA）
  ↓ 命中 → 返回结果 + 来源链接
  ↓ 未命中
第三层：DeepSeek AI 推断（标注为"AI推断，置信度低"）
  ↓ 或
  → 灰色："暂无研究数据，建议咨询医生"
```

## 九、合规要点

### 法规红线

| 能做 | 不能做 |
|------|--------|
| 成分科普解释 | 诊断疾病 |
| 指标趋势展示 | 出具医疗结论 |
| "建议咨询医生" | 替代医嘱的剂量方案 |
| 成分禁忌提醒 | 声称治疗/治愈效果 |
| 来源引用 | 未标注来源的断言 |

### 必须做的

- 每次AI输出强制展示免责声明（UI层，非仅条款）
- 健康数据加密存储，符合《个人信息保护法》
- 体检数据单独授权采集（敏感个人信息）
- 用户数据删除请求必须响应
- "暂无数据"场景不能默认为"安全"，必须标灰

## 十一、风险与应对

| 风险 | 级别 | 应对 |
|------|------|------|
| 数据库覆盖率不足（尤其中药<5%） | 高 | MVP聚焦Top 100处方药×Top 50保健品，覆盖80%场景；中药缺数据时诚实标灰 |
| ChatGPT持续进化，差距缩小 | 中 | 壁垒不在"回答更好"，在"持久化档案+自动交叉+可验证来源"——对话形式做不到 |
| 监管政策变化 | 中 | 定位"健康信息服务"非"医疗服务"，免责声明UI强制展示，不做诊断不做治疗 |
| 用户建档门槛高 | 中 | 选择式输入（预置中国常见药品列表），不让用户打字；后续加拍照识别降低门槛 |
| 安全规则出错 | 高 | 三层架构（硬编码→数据库→AI），高风险规则只用硬编码；灰色>错误红色/绿色 |
| 无融资，资源有限 | 高 | 用AI工具开发；先免费验证需求，再考虑融资 |

## 十二、核心判断

1. **冲突检测不是产品，家庭药柜才是产品。** 检测是ChatGPT也能做的一次性动作；药柜是持久化的系统，是ChatGPT形态上做不到的事。

2. **不跟ChatGPT比"谁分析得好"，跟它比"谁记得住"。** VitaMe的价值不在单次回答的质量，在于持续管理一家人的用药安全。

3. **中立性是核心壁垒。** 不卖货、不推荐品牌、不做广告——这让VitaMe可以说"这个别吃了"，而电商平台和品牌锁定产品永远做不到。

4. **诚实是最大的差异化。** "暂无数据，建议咨询医生"这句话的价值，超过所有花哨的功能。ChatGPT不会说"我不知道"，VitaMe会。

5. **数据飞轮的顺序不能颠倒。** 先有药柜数据→才有用户档案→才有服用记录→才有体检数据→才有归因数据。没有用户的飞轮是空转。
