# 面向普通用户的AI健康保健品Agent产品讨论

### 📑 智能总结

#### 录音信息

- **时长**：约 0小时10分钟

#### 录音总结

本次讨论围绕面向普通用户的AI保健品/健康领域Agent产品展开，明确了产品定位、核心逻辑、技术路径与分工，确定采用小步快跑快速迭代的开发策略。

**当前开发问题复盘**
* **开发循环问题**：之前的开发工作反复推倒重来，陷入无方向的循环中。
* **核心偏差原因**：之前产品方向偏差，主要原因是没有完全明确核心需求与用户路径，偏向了功能化开发。

**产品商业化逻辑**
* **信息差带来商业价值**：因为网络壁垒、ChatGPT部分功能需要付费，很多用户不知道已有大模型相关功能，存在信息差，因此具备商业价值。
* **人群能力差异基础**：大部分普通用户的语言能力和思维能力存在差异，很多用户不知道如何清晰表达需求，也不会配置大模型应用。

**产品定位与用户策略**
* **目标用户定位**：产品面向普通小白用户，而非专业医学生，全部内容需要用通俗语言呈现。
* **垂直细分切入策略**：产品第一阶段采用更专更窄的垂直切入方式，精准打透目标人群，提升用户留存与粘性，后续再进行业务泛化。

**竞品替代逻辑**
* **用户能力门槛说明**：即便用户知道可以自己配置大模型Agent（如Llama+Memory+Skill），大部分普通用户不知道如何安装、配置和使用，甚至不知道该如何提出需求。
* **产品核心价值**：我们的核心是发现用户需求，标准化SOP后组合模型、记忆、技能等模块，打造独一无二的个性化产品，用户使用越多数据越多推荐越精准，用户粘性越高。

**产品冷启动设计要求**
* **冷启动核心要求**：新用户冷启动阶段不能要求用户输入大量隐私信息，否则会直接导致用户流失，属于失败的产品设计。
* **冷启动实现路径**：需要通过多种方式引导用户说出第一句话，后续再引导用户添加吃药提醒等个人信息。

**产品商业化路径**
* **第三方流量引导模式**：产品做一站式健康服务，作为第三方只做流量引导，可以推荐淘宝商品、朋友的线上产品，也可以对接保健品销售代表。
* **价格优势来源**：对接保健品销售代表可以拿到更低的折扣，能为用户提供更优惠的价格，让业务形成正向循环。

**产品技术路径要求**
* **核心技术方向**：产品必须往Agent方向开发，否则没有核心价值。
* **Agent核心构成**：Agent的核心是标准化SOP，加上基于Hermes的提示词、记忆、自我净化能力，后续迭代需要实现自我进化，主动提醒用户，逐步引导用户输入更多信息。

**产品开发方法论**
* **小步快跑快速迭代**：前期无法确定最终成品效果，需要先开发出初步版本，再逐步纠偏优化，发现问题及时调整。
* **用户路径设计要点**：产品核心是设计用户行为路径，从产品开发第一天就要设计好全路径，不能让用户察觉到设计痕迹，就像变魔术一样提前预判用户所有反应。

**个人能力差异讨论**
* **定义问题能力差异来源**：擅长解决问题但不擅长定义问题，差距主要来自知识体系储备、问题抽象能力，以及重点排序意识。
* **核心思考方法**：面对多个待做事项时，要反问自己，如果只做一件事情，那件事情应该是什么。

**开发分工安排**
* **技术对接任务**：需要思考如何参照Hermes的方式整合Agent、相关Skill和API。
* **文档输出任务**：负责将产品SOP落写成正式文档，明确产品目标。

[00:00:29](https://getnotes.seek:29) **产品商业化核心逻辑讨论**

首先回答为什么用户不直接用现有界面，而需要我们做专门的APP：因为网络壁垒的存在，ChatGPT部分功能需要付费，很多用户根本不知道ChatGPT有对应功能，信息差必然带来商业价值。其次，人和人之间的语言能力、思维能力差异极其巨大，多数普通用户不知道该怎么清晰表述需求。

[00:01:21](https://getnotes.seek:81) **产品定位与切入策略确认**

产品面向普通小白用户，而非专业医学生，因此所有内容需要用更通俗的语言呈现。产品采用垂直细分切入策略，不需要做全品类，只需要做更专更窄的方向，这样能精准命中目标人群，获得更高的用户留存和粘性，后续再基于现有用户做业务泛化。

[00:02:40](https://getnotes.seek:160) **自有产品替代自研大模型的逻辑说明**

提出疑问：如果用户自己配置Llama+Memory+Skill也能完成对应任务，我们的产品还有什么价值。讨论明确，现在很多人只知道Llama这个名词，根本不知道怎么给自己配置，不知道安装和使用方法，甚至不知道该怎么在保健品领域问出对应的需求。我们要做的就是发现需求、整理SOP，再组合模型、记忆、技能形成独一无二的个性化产品。

[00:03:42](https://getnotes.seek:222) **产品粘性与冷启动设计要求**

用户使用产品越久，留下的数据越多，推荐就越精准，用户就越离不开产品，这和抖音推荐系统的逻辑一致，是经过验证的真理。产品非常看重冷启动环节，新用户进来不能要求输入大量隐私信息，否则用户会直接流失，属于失败的产品；必须用多种方式引导用户说出第一句话，之后再引导用户添加吃药提醒等个人信息。

[00:04:33](https://getnotes.seek:273) **产品商业化路径梳理**

产品做一站式健康服务，后续可以直接做导购，对接购买渠道。我们作为第三方只做流量引导，可以推荐淘宝商品，也可以推荐朋友的线上产品，还可以对接保健品销售代表。保健品销售代表手里有折扣，我们能帮用户拿到更低的价格，整个业务就能正向运转起来。

[00:05:16](https://getnotes.seek:316) **产品技术路径明确**

产品技术方向一定要往Agent方向做，不然产品没有任何价值。Agent的核心是SOP，加上基于Hermes的提示词、记忆，后续迭代需要实现自我净化、自我进化，能主动提醒用户吃药，帮用户感知用药效果，逐步引导用户输入更多信息，让Agent越来越懂用户。

[00:06:35](https://getnotes.seek:395) **过往开发问题复盘**

双方认可应该尽早做这样的深度沟通，之前不知道能做到什么程度，只能做一点讨论一点，发现问题就改，采用小步快跑的方式，现在已经有了初步的产品感觉，后续只需要做纠偏和优化。之前做出来的效果不好，主要是因为文档写的不够好、过程控制不足，核心是没有完全get到产品的核心重点，开发方向偏向了功能化，而没有聚焦用户。

[00:07:30](https://getnotes.seek:450) **产品核心设计思想分享**

做产品最核心的是用户行为路径，用户路径不是用户自己形成的，而是产品开发第一天就由设计者设计好的。不能让用户察觉到路径是被设计好的，就像变魔术一样，用户的所有情绪反应都要提前预判设计，没有达到效果就是失败的产品。

[00:08:31](https://getnotes.seek:511) **个人能力差异与开发分工安排**

讨论明确，擅长解决问题但不擅长定义问题，差距来自三个方面：知识体系、问题抽象能力、重点排序调整，面对多个事项时要反问自己，只做一件事的话应该做哪件。最后确定分工：一方思考如何参照Hermes整合Agent、Skill和API，另一方负责把SOP落写成文档，明确产品目标，结束本次讨论。
