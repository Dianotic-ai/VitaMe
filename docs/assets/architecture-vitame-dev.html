数据流 · 用户一次 query 全链路

  • 1. 用户在 /query 输自然语言（"我吃鱼油能和华法林一起吃吗"）
  • 2. → POST /api/intent · L0 intakeOrchestrator 依次跑 parseIntent (LLM) → groundMentions → slotResolver → clarify
  • 3. 缺 slot → 返 ClarifyBubble；全齐 → 返 LookupRequest 给前端
  • 4. → POST /api/judgment · L2 judgmentEngine 并发 3 adapter (hardcoded / suppai / ddinter)
  • 5. merger + knowledgeBaseLookup → gray (未收录) / green (已知无风险) / yellow / red 四档
  • 6. → POST /api/translation · L3 translateRisks 单条 LLM 调用 → Zod 校验 → 失败 template fallback
  • 7. → /result render RiskBadge + DisclaimerBlock + DemoBanner
  • ⏱ 预算：60s end-to-end（CLAUDE.md §1.3，D5 从 30s 放宽）

合规红线 · 架构层面强制

  • • §10.3 层级隔离：L2 无 LLM (确定性)；L1 无 adapter imports；L3 不造规则
  • • §11.1 Disclaimer：DisclaimerBlock 每次 AI 响应必带
  • • §11.5 LLM 仅翻译：L3 LLM 不判风险，只把 L2 的 Risk[] 翻成人话
  • • §11.6 Zod 校验：LLM 输出失败 → TemplateFallback
  • • §11.11 DemoBanner：54 条红规则 `pharmacistReviewed: false` → 每响应必挂
  • • §11.12 partialReason：白名单码，adapter.error 不泄到 UI
  • • §11.13 L0 LLM：仅 NER + question phrasing，Zod .strict() 防 level 泄露
  • • §11.4 SourceRef：每条 Risk 必带 SourceRef[]

  • • 主链：minimax /anthropic + MiniMax-M2.7
