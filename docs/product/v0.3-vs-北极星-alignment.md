# v0.3 vs Agent 北极星 — Alignment Report

> **目的**：partner 4/26 测试 v0.3 后质疑"用户流程和北极星指标是什么"。本文回答 — v0.3 实际产品形态对照北极星 9 节，标注现状 + 给出 P1/P2 接力路径。
> **结论摘要**：v0.3 不是偏离北极星，是北极星 §9 设计的"P0 預埋"阶段提前选了对话化形态。P1/P2 已规划，将在 v0.4 分支实施。

---

## 0. 背景 — 为什么 v0.3 看起来跟北极星不像

### 0.1 v0.2 vs v0.3 pivot 时间线

| 时间 | 状态 |
|---|---|
| 2026-04-16 ~ 2026-04-24（v0.2） | 严格按北极星 P0 spec 实现：4 层流水线（L0 intent → L2 deterministic → L3 template）+ archive 保存 + recheck 入口 |
| 2026-04-25（pivot 决策） | Sunny 测试 v0.2 后判断："对话效果不如 ChatGPT/Claude/Gemini，WAIC demo 力度不够"。拍板 pivot v0.3 RAG chatbot |
| 2026-04-25 ~ 2026-04-26（v0.3） | 重写为对话式 RAG chatbot：流式 streamText + KB 检索 + 多人健康档案 |

## 2. 北极星 §2 Core Loop

```
加入一款补剂 → 属性识别 → 设置提醒 → 服用后轻反馈 → Memory 沉淀 → Hermit 周期归纳 → 用户确认观察 → 调整提醒或复查
```

| 环节 | v0.3 现状 |
|---|---|
| 加入一款补剂 | ⚠️ 用户可在对话里说"我在吃 X"，但**没有结构化"加补剂"入口** → P1 D2 加 |
| 属性识别 | ✅ KB retriever 自动从 8 源识别成分属性（剂量 / 形式 / 相互作用）|
| 设置提醒 | ❌ → P1 D4 加 |
| 服用后轻反馈 | ❌ → P1 D3 加 Feedback Ritual |
| Memory 沉淀 | ⚠️ profileStore 沉淀 condition/medication/allergy/conversationSummary，但缺 §5 MemoryEvent 时间流 → P1 D1 升级 |
| Hermit 周期归纳 | ❌ → P2 D6-7 加 |
| 用户确认观察 | ❌ → P2 D6-7 加 observation card |
| 调整提醒或复查 | ❌ → P2 D7 加 |

**结论**：v0.3 完成了 Core Loop 的"入口"（属性识别）+ "Memory 雏形"，缺中间所有触发器和闭环。

## 3. §3 Reminder

| 北极星要求 | v0.3 现状 | P1 D4 计划 |
|---|---|---|
| 帮用户减少忘记服用 | ❌ | 浏览器 in-app banner（用户访问时检查到点的 reminder）|
| 在合适时机触发轻反馈 | ❌ | banner click → FeedbackPrompt 模态 |
| 默认低频 | ❌ | 默认每天 1 次（用户可改）|
| 用户可暂停/跳过/降频 | ❌ | reminder rule 加 pause / skip / lower-frequency 字段 |
| 连续无响应自动降频 | ❌ | scheduler 检测连续 3 次 no-response → 频率减半 |
| 高风险补剂不生成"放心服用"暗示 | ✅ chat 层已有 critical 硬路由 | 复用规则 |

## 9. §9 Relationship to P0

> **北极星原文**："P0 不实现完整 Agent 北极星，只预埋保存和复查。P1 才开始 Reminder + Feedback，P2 才开始 Memory + Hermit。"

### 9.1 v0.3 是不是符合 §9

**v0.3 没严格按"保存和复查"形态做 P0**，而是选了"对话化"形态。两者都是"預埋" — v0.2 預埋 archive 数据，v0.3 預埋 profile + conversation history。

**等价性论证**：
- v0.2 archive：用户保存一次安全检查结果 → P1 reminder 基于 archive 触发
- v0.3 profile + conversationSummaries：对话沉淀疾病 / 用药 / 历史摘要 → P1 reminder 基于 profile + 加补剂入口触发

**核心数据都在 LocalStorage**，P1/P2 接力路径并未被破坏。

### 9.2 P1 接力计划（v0.4-northstar-loop 分支，5/1+ 实施）

| Day | 任务 | 验收 |
|---|---|---|
| **D1** | MemoryEvent schema + store + 时间轴 | partner 看 /memory 时间轴 |
| **D2** | 加补剂入口 + currentSupplements 字段 | partner 在 /profile 加一款补剂 |
| **D3** | Feedback Ritual 模态 | partner 重访 24h 后看到反馈弹窗 |
| **D4** | Reminder in-app banner | partner 设提醒后到点看到 banner |
| **D5** | "LLM 看到什么"视图 | partner 点击查看注入的 XML scaffold |

### 9.3 P2 接力计划

| Day | 任务 | 验收 |
|---|---|---|
| **D6-7** | Hermit MVP（Vercel Cron + 后台 LLM 归纳 + observation card）| partner 一周后看到主动观察建议 |

## 11. 一个诚实的注脚

WAIC 演示用 v0.3，是因为对话 demo 比"安全检查 + 保存"对评委更有视觉冲击力。但**北极星才是产品长期价值**。WAIC 之后 v0.3 形态本身可能要往北极星方向再演化（比如"对话产生 archive 自动保存 → reminder 跟踪"），不是简单加 P1 模块。

partner 的角色在这个阶段是**保持北极星方向不漂移**。Sunny 的角色是**短期为获奖做战术调整**。两者不冲突，但需要这种 alignment 文档把视角对齐。
