# VitaMe — 留存飞轮

## 1. 核心判断

VitaMe 不是靠“每天来聊”留存，而是靠健康事件触发的高价值闭环留存。

长期飞轮：

```
安全判断 → 保存 → 提醒 → 轻反馈 → Memory → Hermit 观察 → 调整提醒/复查 → 下一次更准
```

## 2. P0 飞轮

外部事件触发 → 用户查询 → VitaMe 给出风险判断和解释 → 用户保存 → 下次新增一项时快速复查。

成功条件：

- 第一次判断可信。
- 保存能减少下一次输入。
- 用户形成“先查再买”的习惯。

## 3. P1 飞轮

保存结果 → 一键确认提醒 → 到点提醒 → 用户反馈是否服用和感受 → 上下文自然长出来。

成功条件：

- 提醒不打扰。
- 反馈不重。
- 每个问题都有即时价值。

## 4. P2 飞轮

连续反馈 → Memory 累积 → Hermit 周期扫描 → 生成可解释观察 → 用户确认/否定 → Agent 调整后续提醒或复查策略。

成功条件：

- 观察来自用户自己的数据。
- 用户能看见证据。
- 用户能纠错。

## 5. Growth Loop

高风险判断 → 可理解结果 → 家庭成员分享 → 保存家人档案 → 提醒和反馈 → 更完整上下文 → 更可信结果 → 更多家庭复查。

## 6. Guardrails

- 不为了 DAU 制造提醒。
- 不把反馈做成症状诊断。
- 不把 Hermit 观察写成医学因果。
- 不用导购污染信任。
- 不隐藏 Memory 和 LLM 输入。

## 7. Metrics

| Phase | Retention Metric | Guardrail |
|---|---|---|
| P0 | 30 天二次查询率 | 灰色不得伪装绿色 |
| P1 | 提醒后反馈率 | 退订/降频率 |
| P2 | 确认观察率 | 用户否定率和删除率 |

## 8. Conclusion

VitaMe 的留存不是提醒用户“回来用 app”，而是让每一次健康事件都把 Agent 训练得更适合这个人。
