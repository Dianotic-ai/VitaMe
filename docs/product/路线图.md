# VitaMe — Outcome Roadmap

## 1. Roadmap Principle

路线图不按功能堆叠，而按用户结果排序：用户先完成安全判断，再愿意保存，再接受提醒和反馈，最后让 Agent 基于长期 Memory 自我进化。

## 2. Phases

| Phase | Outcome Statement | Key Metric | Primary Outputs |
|---|---|---|---|
| P0 | Enable supplement buyers to make a trustworthy safety decision before purchase or use so that VitaMe earns first-use trust. | Qualified Safety Decision | L0/L2/L3 主链、结果页、免责声明、DemoBanner |
| P1 | Enable users to turn a saved safety result into a lightweight reminder and feedback loop so that context grows without forms. | Reminder feedback rate | Reminder table、Feedback Ritual、privacy HUD v0 |
| P2 | Enable the Agent to identify user-specific patterns from Memory so that recommendations become more personal without diagnosis. | Confirmed Hermit observation rate | Memory schema、recall、Hermit cycle、observation cards |
| P3 | Enable users to add more health events into the same loop so that VitaMe becomes a general personal health context engine. | Multi-event active users | medication/sleep/diet/exercise/checkup event adapters |

## 3. P0 — 能查，能信

Customer outcome：用户能在 60 秒内知道一款补剂是否需要警惕、为什么、下一步怎么做。

Must ship：

- Natural language intent → grounded lookup。
- 红黄灰绿判定。
- 人话解释和证据。
- 免责声明和未审核规则提示。
- 结果保存入口。

Do not ship：

- 完整 Reminder 引擎。
- Memory/Hermit。
- 导购和 CPS。

## 4. P1 — 能提醒，能反馈

Customer outcome：用户保存后能一键设置提醒，并用低摩擦反馈让上下文自然生长。

Must ship：

- 成分属性查表：水溶/脂溶、时段、随餐/空腹、禁忌提示。
- Reminder create / pause / adjust。
- Feedback Ritual 文案库和严重关键词拦截。
- 隐私 HUD v0：用户知道哪些字段进入 LLM。

## 5. P2 — 能记住，能进化

Customer outcome：用户看到 Agent 基于自己的真实反馈提出可解释观察。

Must ship：

- Memory schema。
- Top N 相关 Memory 召回。
- Hermit 周期任务。
- Observation card：证据、置信度、用户确认/否定。
- 用户可删除或关闭某类 Memory。

## 6. P3 — 能迁移

Customer outcome：用户能把更多健康事件放入同一套 Agent 反馈循环。

Candidate events：

- 处方药。
- 睡眠。
- 饮食。
- 运动。
- 体检报告。

## 7. Strategic Dependencies

- P1 不能在 P0 可信前启动，否则提醒只会放大不信任。
- P2 不能在 P1 有稳定反馈前启动，否则 Hermit 没有可学习数据。
- P3 不能在隐私 HUD 和 Memory 控制权成熟前启动，否则健康数据敏感度不可控。

## 8. Trade-offs

- 不做电商导购，换取信任。
- 不做默认完整建档，换取冷启动转化。
- 不做开放式诊断，换取合规边界。
- 不做不可解释自动建议，换取用户主权。
