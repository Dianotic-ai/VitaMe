## 二、优先级分层

### P0 — 不做产品不成立

| 功能 | 理由 |
|------|------|
| A1 药物-保健品冲突检测 | 唯一的恐惧驱动获客入口，产品定位的锚点 |
| B1 AI健康对话 | 用户查完一定追问"为什么"，无对话=死产品 |
| G1 免责声明系统 | 没有=法律自杀 |
| G2 数据源透明 | 没有=零信任 |
| G3 医学顾问背书 | 一行字的信任价值超过所有代码 |

### P1 — 做了体验质变

| 功能 | 理由 |
|------|------|
| A2 保健品-保健品冲突 | 用户吃3种以上保健品很常见，只查药物不够 |
| A3 病史禁忌校验 | 肾结石不能补钙——这类场景缺了就不完整 |
| B2 安全替代推荐 | "不能吃"之后必须告诉"该吃什么"，否则价值断裂 |
| B3 服用时间建议 | 低成本高感知价值，结构化数据即可 |
| C1 家庭保健品档案 | 留存基础，不建档=每次重新输入=流失 |
| H1 社交裂变卡片 | 获客引擎，冲突检测天然适合分享 |

### P2 — 有加分不影响核心

| 功能 | 理由 |
|------|------|
| D1 拍照识别 | 降低输入门槛但OCR难度大，手动输入先行 |
| F1 多平台价格参考 | 省钱钩子，但需持续维护数据 |
| F2 选购内容聚合 | 内容运营依赖持续投入 |
| C4 服用打卡 | 单独无价值，但为E1积累数据 |
| C3 远程督药 | 情感故事极强，但微信推送有硬限制 |
| C2 家庭组 | C1做好后的自然延伸 |

### P3 — 远期愿景

| 功能 | 理由 |
|------|------|
| D2 体检报告解读 | 医院格式不统一，技术难度★★★★★ |
| D3 体检纵向趋势 | 需2年数据积累，0-1阶段是自杀 |
| E1 效果关联展示 | 全球空白=终极壁垒，但需C4+D2数据先行 |
| E2 方案智能调整 | 闭环终局，需E1成熟 |
| F3 跨境成分翻译 | 海淘用户场景，非核心 |
| H2 月度健康简报 | 留存工具，需数据基础 |

| 功能 | 状态 | 说明 |

## 四、阶段规划

### MVP（14天） — 主题："能查，能信"

- **核心交付**：冲突检测可用+AI对话可用+信任体系完整
- **解锁能力**：用户可以输入药物+保健品+病史，30秒获得安全评估
- **关键指标**：Demo完成率、现场观众"我也想查"的反应
- **做实**：A1/A2/A3 + B1/B3 + G1/G2/G3
- **模拟展示**：B2 + C1 + H1 + E1

### V1上线（MVP后1-2个月） — 主题："能存，能传"

- **核心交付**：家庭档案真实可用+社交裂变上线+替代推荐动态化
- **解锁能力**：用户建档后不再重复输入；分享卡片带新用户
- **关键指标**：7日留存率、分享率、人均建档成员数
- **新增做实**：C1家庭档案 + H1裂变卡片 + B2动态替代推荐
- **技术重点**：Supabase用户体系+微信H5分享+成分知识库扩展至200条

### V2（3-6个月） — 主题："能管，能省"

- **核心交付**：服用管理闭环+选购辅助
- **解锁能力**：用户从"查一次"变成"每天用"（打卡+提醒+比价）
- **关键指标**：DAU/MAU比、打卡坚持率、订阅转化率
- **新增**：C4打卡 + C3远程督药 + C2家庭组 + F1价格参考 + D1拍照识别
- **商业化**：订阅层上线（¥9.9-19.9/月，解锁深度报告+家庭组+历史记录）
- **技术重点**：微信小程序上线（ICP已就位）、OCR集成、推送机制

### V3（6-12个月） — 主题："能追，能调"

- **核心交付**：效果追踪+方案调整——全球无竞品的终极壁垒
- **解锁能力**：用户上传前后体检→系统关联展示"吃了X→指标Y变化"→建议调整
- **关键指标**：体检上传率、追踪完成闭环率、B2B签约数
- **新增**：D2体检报告解读 + E1效果关联展示 + E2方案调整 + H2月度简报
- **商业化**：B2B企业健康管理上线（¥50-500/员工/年）
- **技术重点**：体检报告OCR+结构化（先支持Top 10连锁体检机构格式）

## 五、战略节奏总结

```
用户感知层:  查安全 ──→ 查安全+存档案 ──→ 管家人 ──→ 追效果
             (一直是"查安全"的延伸，用户不需要重新认知)

产品实现层:  检测引擎 → 用户体系+增长 → 管理闭环+商业化 → 数据壁垒+B2B
             MVP          V1              V2               V3

数据飞轮:   规则库 → 用户档案 → 服用记录 → 体检数据 → 归因数据
            (每层数据为下层功能提供燃料，竞争对手无法跳步复制)
```

**核心判断**：冲突检测是钉子，用恐惧驱动获客；家庭档案是粘合剂，用迁移成本留存；效果追踪是护城河，用独占数据建壁垒。三者的时间顺序不能颠倒。
