# VitaMe — 产品定位

## 1. 一句话定位

**VitaMe 是一个从补剂安全判断开始、能通过提醒和反馈逐渐懂你的健康 Agent。**

它先回答“这款补剂对我是否需要警惕、为什么、该避开什么”，再通过保存、提醒、反馈和 Memory，把一次判断变成长期可复查、可学习的个人健康上下文。

## 2. 电梯版

普通用户不是缺健康内容，而是缺一个能把补剂、病史、在用药和真实反馈连起来的判断系统。

VitaMe 的第一步是安全排雷：买前先查。第二步是形成闭环：查完保存、提醒服用、记录感受、让 Agent 逐渐理解用户自己的反应模式。

## 3. 产品不是谁

- 不是疾病诊断工具。
- 不是保健品导购或 CPS 平台。
- 不是无限聊天的健康顾问。
- 不是要求用户先填完整档案的后台系统。
- 不是医学结论或处方建议生成器。

## 4. 产品是谁

- P0：补剂安全判断工具。
- P1：提醒 + 反馈闭环。
- P2：Memory 驱动的自我进化 Agent。
- P3：可扩展到处方药、睡眠、饮食、运动、体检的大健康事件管道。

## 5. 核心用户

### 首发用户

准备购买、开始服用、替换或追加补剂的人。他们已经有具体对象，不想读长篇科普，只想知道是否需要警惕。

### 高价值用户

有病史、在用药、孕期、老人照护、长期多补剂叠加等上下文的人。他们的风险不是通用内容能解决的。

### 长期用户

愿意让 VitaMe 帮自己或家人记录服用、反馈和复查的人。他们不是每天来聊天，而是在每次健康事件发生时让 Agent 变得更懂自己。

### P0 Job

当我准备买或吃一款补剂时，我想快速知道它对我是否有已知风险、为什么、下一步怎么更稳妥。

### P1 Job

当我决定开始服用一款补剂时，我想让系统帮我安排提醒，并用最少动作记录是否服用和有什么感觉。

### P2 Job

当我已经持续反馈一段时间后，我希望 Agent 能发现我自己看不到的模式，并用可确认、可撤销的方式提醒我调整。

## 7. 信任主张

- 有风险直说。
- 没证据也直说。
- 不卖货。
- 不诊断。
- 不把 LLM 当医生。
- 让用户看见信息如何被使用。

## 8. 当前边界

P0 必须保持窄：补剂安全判断、解释、保存。Reminder、Feedback、Memory、Hermit Agent 是长期北极星，不得被写成当前已实现能力。

## 9. 定位结论

**VitaMe 先用补剂安全判断获得信任，再用提醒和反馈获得持续上下文，最终让 Agent 在用户的真实生活反馈中自我进化。**
