## 架构决策（我替你做的判断 + 理由）

### 1. 单次 streamText（不用 ToolLoopAgent）
- 我们之前算过 ToolLoopAgent 9 次 LLM 在 Vercel Hobby 60s 上限内装不下
- 改用单次 streamText + retriever 注入 facts，Vercel 60s 完全够用
- v0.2 的 /api/agent 路由保留不动，仍然可以访问，只是 chat 路径不调用

### 2. Profile 客户端 LocalStorage（不上服务端）
- CLAUDE.md §9.8 明文要求"仅本地"
- 客户端每次 POST /api/chat 时把 profile 当 body 传一下
- 服务端只在 prompt 里用，不持久化
- 一键清空按钮在 /profile 页面

### 3. Memory extractor 用同一 minimax 国际版（轻量）
- 不另起 LLM；同一 endpoint 跑 generateText（temperature=0）
- 抽取在主 chat 流结束后异步触发（fire-and-forget）
- 失败/超时不影响用户感知（用户已经看到完整回复）

### 4. RAG retriever 用关键词 grep（不引向量库）
- 4 天 deadline 内不引 pgvector / pinecone
- 别名表 ~500 entries（中英文 + 常用别名 + 8 源 substance.id）覆盖 v0.3 测试场景
- 每源最多 10 条，避免 prompt 爆炸（CLAUDE.md §14 坑 #2）
- 未来升级路径：把 keyword grep 换成 embedding similarity（接口签名不变）

### 5. 老 v0.2 代码全部物理保留
- 5 个老路由 + 5 个 capabilities + 5 个老 page，全部不动
- v0.3 chat 路径 0 import 老代码（除了 L1 db 数据，那是共享的）
- 任何时候 `git checkout v0.2` 秒回滚到 v2.10 严格架构

## ⚠️ 已知开放问题（你早上要拍板）

### 🟡 P1 — chat 体验类
1. **LLM 偶尔创造性使用引证标记**：测 Q10 多轮时看到 `[来源: 一般认知，retrieved_facts 未收录此相互作用]` — 这违反引证只能用于真实源的规则。**修法**：system prompt 加一条明文"[来源: ...] 标记仅可用于 retrieved_facts 中真实存在的 source name"。
2. **首屏空状态**："跟 VitaMe 聊聊你最近想买的补剂" + 2 个示例。要不要加更多示例 chip 让用户一键发？(你之前说删掉旧的 6 个示例，所以我放得很克制。)
3. **流式时禁词后置校验生效但不撤回**：banned word 命中只写 audit，不能"撤回"已流出的 token。CLAUDE.md §9.6 文字是"硬错误"，但流场景不可能。我的妥协是：写 `chat_banned_word_hit` audit 事件，未来可统计频率。**你 OK 吗？**
4. **多轮历史 token 增长**：当前 useChat 把全 history 传给 backend，5+ 轮后 prompt 会变大。CLAUDE.md §14 坑 #6 说"仅保留最近 5 轮"。**目前没做截断**。要不要补？

### 🟢 P2 — UX polish
5. **chat 页面没有"新对话"按钮**：useChat 默认 in-memory，刷新就丢。如果要跨刷新保留对话，需要把 messages 也存到 zustand。**P0 我先没做**（CLAUDE.md §2.3 没列），但你可能想要。
6. **profile 页面没"导出/导入"**：用户换设备就丢档案。LocalStorage 限制。如果要跨设备，必须做服务端用户系统（你说 P0 不做）。
7. **chat 输入框的 placeholder 文案**：当前是"聊聊你想了解的补剂…"，可以更具体。
8. **Markdown 渲染**：用了 react-markdown + remark-gfm，但未做 syntax highlight（用户问代码不太可能，但 v0.3 没装）。

### 🔴 阻塞 / 决策点
9. **vitame.live 域名 CNAME 还没切**（T-D10.2）：现在 demo URL 是 https://vita-me-ebon.vercel.app/chat。要不要现在切？
10. **Kevin 流程文档**：你说今天会给一份完整 user flow（体检报告 + 多产品分阶段）。这会影响 prompt 第二轮迭代和 profile schema 是否要加 `current_supplements` 字段。
