## 2. 矛盾点清单

### 2.1 多 person 档案

- **action-first 文档**：「MVP 不做多人档案」「首访不能有 PersonSwitcher」「测试可覆盖『我妈在吃 X』但不能物化为产品功能」
- **v0.4 实际**：完整 multi-person profile（profileStore.people[]）+ chat header PersonSwitcher + /profile 多 person 管理 + 跨 person memory/reminder 隔离

### 2.2 Profile 注入 chat

- **action-first 文档**：「`/api/chat` body 只许 `{sessionId, messages}`」「不能携 profile / userMentioned / healthProfile」
- **v0.4 实际**：`/api/chat` body 含 `profile: ProfileSnapshot`（conditions/medications/allergies/specialGroups/currentSupplements/ageRange/sex/recentTopics 全注入 system prompt 的 `<user_profile>` XML）

### 2.3 `/api/extract` 调用

- **action-first 文档**：「不允许每轮对话后调用 LLM 抽取健康档案」「Network 面板看不到 `/api/extract`」
- **v0.4 实际**：每轮 chat assistant 完成后 fire `/api/extract`，LLM 抽取 newConditions / newMedications / newSupplements / newAllergies / ageRange / sex / conversationSummary 写入 profile

### 2.4 Memory 自动写

- **action-first 文档**：「Memory 必须事件式」「只在用户**显式动作**后写（routine_saved / product_judgment_saved / feedback_recorded / safety_user_declared）」
- **v0.4 实际**：每轮 chat 自动写 verify event；reminder ack / feedback prompt 提交 / 设规则 / Hermit accept-dismiss 都写 event

### 2.5 Hermit Agent

- **action-first 文档**：「Hermit 自动周期归纳」属于「暂缓资产，不进入 P0 主路径」
- **v0.4 实际**：/memory 页有 HermitButton（手动触发）→ /api/hermit 调 LLM → 写 observation event 到时间轴

### 2.6 Reminder 中心

- **action-first 文档**：「Reminder 中心页的复杂规则管理」属于「暂缓资产」
- **v0.4 实际**：完整 `/reminders` 页（今日时间表 + 全部规则 + 暂停/降频/skip 自动降频）

### 2.7 Feedback ritual

- **action-first 文档**：「Feedback 弹窗自动打扰」属于「暂缓资产」
- **v0.4 实际**：进 /chat 1.5s 后检查 trigger，符合条件自动弹 FeedbackPrompt

### 2.8 Product Inspect / Routine 概念

- **action-first 文档**：要求做 Product Inspect URL 解析 + RoutineConfirmCard + 4-slot 早/中/晚/睡前 routine
- **v0.4 实际**：无 Product Inspect；ReminderRule 是任意 timeOfDay，不是 4-slot routine。**但 v0.4 D14 PillBox** 已按 4-slot 视觉契约渲染（rule.timeOfDay 在渲染时 bucket 进 4 格）

## 4. 三种解决路径（5 月之后选）

### 路 A：按 action-first 重写主路径（产品方向重写）

- 拆 PersonSwitcher / /api/extract / profile 注入
- 引入 Product Inspect + Routine + 6 段式回答
- v0.4 已实现的 Memory 时间轴 / Hermit / Reminder 中心 / Feedback ritual 全转"暂缓资产"（git tag 冷藏）
- 工作量：1-2 周
- 适合：Kevin 想把产品收回更聚焦 MVP

### 路 B：v0.4 是新北极星，重写 action-first 文档

- 多 person + 长期记忆 + 自动归纳 = 产品差异化
- action-first 文档反映过时判断，按 v0.4 实际重写
- 工作量：纯文档，半天
- 适合：partner / 评委反馈"v0.4 完整 loop 是亮点"

### 路 C：双轨制（两条产品并存）

- main 保留 v0.4 完整 demo 体验
- 另开 `v0.5-action-first` 分支按 action-first 重做精简版
- 两条路并跑，市场验证哪条用户接受度高
- 工作量：路 A + 维护 v0.4 demo 状态
- 适合：还没想清楚，先各做各的看反馈
