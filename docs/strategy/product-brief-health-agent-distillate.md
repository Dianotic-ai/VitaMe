# 产品简报蒸馏包: 健康守护Agent

> 状态：已过期。
> 本文是早期产品简报的蒸馏稿，继承了“体检趋势分析 + 微信小程序”的旧前提，不再代表当前产品方向。
> 请优先参考：`docs/product/VitaMe-补剂安全翻译Agent-定位.md`、`docs/product/VitaMe-补剂安全翻译Agent-P0-PRD.md`

## 竞品格局

### 直接竞品定位
- **丁香医生**：医疗科普+在线问诊，无体检趋势/保健品成分级解析 → 偏医疗端
- **薄荷健康**：饮食热量管理，无保健品模块 → 聚焦减脂
- **妙健康**：B端慢病管理，C端弱 → 无AI对话
- **华为/小米健康**：穿戴设备+运动监测，保健品领域空白 → 有数据无解读
- **Examine.com**：学术级成分数据库(英文,$29.99/月) → 无个性化推荐，无中文化
- **Ada Health**：AI症状检查 → 无保健品，未进中国

### 市场空白（核心差异化）
- 无产品做"多年体检纵向趋势解读"（全是单次快照）
- 保健品成分中文循证解析几乎空白
- "病史×成分"禁忌交叉校验无系统化产品
- "看懂体检→选对保健品→坚持吃"三段串联无竞品

### 市场规模
- 中国保健品市场 ~3500亿元(2025)，年增8-10%
- 数字健康管理 ~8000亿元(2025)
- 亚健康人群 >9亿，健康焦虑年轻化

## 成分知识体系要点

### 成分形式差异（核心差异化功能数据）
- 镁：氧化镁(吸收率~4%,便宜) → 甘氨酸镁(高吸收,助眠) → 苏糖酸镁(穿透血脑屏障)
- 铁：硫酸亚铁(刺激胃) → 甘氨酸亚铁(温和) → 铁蛋白(极温和)
- 钙：柠檬酸钙(不依赖胃酸) → 碳酸钙(需随餐)
- B12：氰钴胺(需转化) → 甲钴胺(活性形式,MTHFR突变首选)
- CoQ10：泛醌(需转化) → 泛醇(还原型,40岁+首选)

### 关键禁忌（安全规则引擎必须硬编码）
- 肾结石(草酸钙) → 避免高剂量维C(>1g)、钙补充剂
- 肾功能不全 → 避免镁、钾补充剂
- 服华法林 → 避免维K、高剂量鱼油、银杏
- 甲亢 → 避免含碘补充剂
- 肝病 → 避免高剂量维A、卡瓦
- 孕期 → 避免维A(>3000IU)、某些草本

### 服用时间规则
- 脂溶性维生素(A/D/E/K) → 随含脂肪餐
- 铁剂 → 空腹+维C，避免与钙/茶同服(间隔2h+)
- 镁 → 晚间(助眠)，避免与铁同服
- 益生菌 → 空腹或餐前30分钟

## 体检分析要点

### 预警逻辑
- 方向性变化：连续2-3年同向变化（即使在正常范围内）
- 变化速率：短期剧烈 vs 缓慢漂移 → 不同预警级别
- 关联分析：多指标联动(MCV↑+B12↓=巨幼贫倾向)

### 关键指标-保健品关联
- 血红蛋白↓ → 铁/B12/叶酸
- ALT/AST↑ → 审查在服补充剂
- eGFR↓ → 调整含镁/钾推荐
- LDL-C↑ → Omega-3窗口
- 空腹血糖升 → 铬/肉桂提取物辅助
- TSH异常 → 碘/硒策略调整
- T值下降 → 钙+D3+K2组合

## 法规合规要点

### 能做 vs 不能做
- ✅ 成分科普解释、指标趋势展示、"建议咨询医生"、成分禁忌提醒、比价信息
- ❌ 诊断疾病、出具医疗结论、替代医嘱的剂量方案、声称治疗/治愈效果

### 免责声明
- 每次输出必须包含：本系统不提供医疗诊断；建议仅供参考不替代专业医疗意见；有疾病请遵医嘱
- UI层强制展示，非仅在条款中

### "循证"风险
- 保健品领域高质量RCT极少
- 需明确数据源(DSLD/PubChem)和证据分级标准
- 证据不足时标注"有限证据"而非断言

## 技术数据源

| 来源 | 用途 | 获取方式 |
|------|------|---------|
| DSLD(美国膳食补充剂标签库) | 成分-剂量映射 | 公开API |
| PubChem/ChEBI | 化学形式标准化 | 公开 |
| 中国保健食品注册数据库 | 蓝帽子产品信息 | 公开 |
| Natural Medicines Database | 成分-疗效-禁忌 | 付费 |
| iHerb/Amazon产品页 | 成分标签/价格 | 爬取 |

**MVP建议**：先手动整理Top 50高频成分知识库(覆盖~80%场景)

## 审查发现（需用户决策）

### 怀疑者审查 — 关键质疑
1. **冷启动问题**：首次用户只有1份报告，体验不到"纵向趋势"核心卖点 → 已采纳：加"单次报告解读+同龄人对比"即时价值
2. **打卡无壁垒**：红海功能，分散资源 → 已采纳：Demo阶段砍掉或极简化
3. **CPS佣金与中立性冲突** → 已采纳：推荐算法与佣金解耦+透明披露
4. **Demo范围风险**：两周全链路不现实 → 已采纳：明确demo只演示核心链路
5. **付费意愿未验证**：深度报告¥9.9是否有人买？ → 待验证，建议landing page测试

### 机会发现 — 被低估的机会
1. **叙事升级**："提前3年发现健康拐点" >> "看懂体检" → 已采纳
2. **成分冲突检测独立入口**：裂变传播点 → 已采纳
3. **家庭组功能**：宝妈管全家报告 → 列入后续迭代
4. **企业团检B端渠道**：美年/爱康合作批量获客 → 列入后续迭代
5. **跨境购信任链**：拍照识别海外成分表 → 列入后续迭代
6. **打卡数据反哺推荐**：依从性数据是品牌方付费金矿 → 列入长期商业模式
7. **中医体质辨识融合** → 列入后续迭代
8. **微信社交裂变**：家人健康摘要卡片分享 → 列入后续迭代

## 被否决的方向
- MVP阶段不做OCR（手动录入足够验证核心价值）
- MVP阶段不做比价爬虫（合规风险+开发量大）
- 不做穿戴设备（需要硬件合作，与黑客松时间冲突）
- 打卡功能Demo阶段极简化（聚焦核心链路）

## 开放问题
- 付费意愿是否成立？深度报告 ¥9.9 的定价是否合理？
- Hermes Agent框架能否满足微信小程序部署需求？技术集成复杂度？
- Top 50成分知识库的整理工作量？是否有现成数据可加速？
- 黑客松评审标准是什么？Demo重点应偏技术演示还是用户体验？
