## Problem Statement

中国9亿亚健康人群面临三个断裂的健康决策环节：看不懂体检报告趋势、不知道保健品成分差异（氧化镁vs甘氨酸镁吸收率差10倍）、不清楚个人病史禁忌（肾结石患者不能高剂量维C）。现有产品要么做体检解读（蚂蚁阿福、讯飞晓医、京东健康——已是红海），要么做品牌级营养包（NUTRIPACKS橘袋），没有产品在"成分化学形式级"做个性化推荐+禁忌交叉校验。

创始人的搭档亲历此痛点：上传10年体检报告、看了多个保健品视频，仍然无法判断成分差异和禁忌关系。

## Demand Evidence

- 创始人搭档的真实经历：对比京东和iHerb同款产品价差一倍，无法理解英文成分名，不知道自己的结石病史与哪些成分冲突
- 蚂蚁阿福（Ant Health）3000万月活但**明确不做保健品推荐**——大厂主动回避 = 需求真实但风险也真实
- PRFAQ概念评分7/10，最大优势：真实空白市场、差异化锐利
- 中国保健品市场2023年零售额2253亿元，同比增长11.6%
- AI个性化营养全球市场：USD 1.59B (2025) → USD 17.72B (2035)，CAGR 27.4%

## Status Quo

用户当前解决方案：
1. **体检报告**：医生说"还行"就放一边，从不做纵向对比
2. **保健品选购**：百度搜索/小红书种草/朋友推荐，按品牌而非成分选择
3. **禁忌检查**：不检查。靠运气。或者完全不知道存在禁忌关系
4. **服用管理**：买了忘吃、吃错时间、坚持不了

核心洞察：**"什么都不做"是最大的竞争对手**。大多数人的解决方案就是忽略问题。

## Target User & Narrowest Wedge

**Target User:** 25-35岁亚健康白领，久坐办公，身体开始发信号（腰痛、眼干、肠胃不适），有改善意愿但信息不对称。典型画像：28岁程序员小王，刚做完年度体检，想改善但不知从何入手。

**Narrowest Wedge:** "成分冲突检测器"——用户输入正在吃的2-5种保健品+病史，30秒返回红/黄/绿冲突结果。这是最小可验证的价值单元：
- 不需要上传体检报告（降低门槛）
- 高频场景（每次买新保健品都可用）
- 天然社交裂变（"你吃的这几样冲突吗？"适合微信分享）
- 可独立变现（单次检测可收费或引流）

## Constraints

- **时间**：17天内完成，4月30日WAIC OPC报名截止
- **团队**：2人（Sunny + 搭档）
- **预算**：月运营成本不超过¥500
- **合规**：定位"健康信息服务"非"医疗服务"，小程序类目"生活服务-营养查询"
- **安全红线**：禁忌规则硬编码不依赖LLM，严重异常强制就医，免责声明UI层强制展示
- **技术**：DeepSeek-V3 API + FastAPI + uni-app + SQLite + 腾讯云

## Premises

1. **核心差异化 = "成分形式级推荐 + 禁忌安全引擎"**（刀尖），三合一串联作为完整故事（刀身）。体检分析和打卡已是红海，不再作为主要卖点。
2. **"成分冲突检测"是Demo主角之一**，与AI对话并列展示，不是附属功能。它是高频留存钩子（vs体检1年1次的低频）。
3. **Demo先建立安全感**——用冲突检测器抓住一个可怕的禁忌案例制造"moment"，再展示完整AI对话能力。CPS商业模式口述，不在产品中展示。
4. **合规包裹的推荐引擎才是产品本身**——禁忌规则、强制就医提醒、成分证据引用不是功能，是护城河。蚂蚁阿福不碰这个领域 = 安全基础设施是核心壁垒。
5. **健康App 30天留存率只有4%**——必须有高频使用场景（冲突检测）而非仅依赖低频场景（体检分析）。

## Cross-Model Perspective

独立Claude子代理的冷读分析（完整原文保留）：

**Steelman:** 这是一个"安全优先的保健品智能层"，建在商品化体检分析之上。体检分析是漏斗，保健品推荐是护城河，冲突检测是留存钩子。

**最关键数据点:** 蚂蚁阿福3000万月活但明确不做保健品推荐。当一个有蚂蚁集团撑腰的公司主动回避你的核心功能，说明合规/责任风险是真实的——你的真正产品不是AI聊天机器人，而是一个"合规包裹的推荐引擎"。

**挑战的前提:** "Demo先展示推荐精确度来建立信任"——在黑客松里这是反的。评委无法验证精确度，但能在内脏层面感受到"避开了一个危险"。Demo应该先用冲突检测抓住可怕的禁忌案例。恐惧比信任更有力。

**48小时原型建议:** 只做冲突检测单页，跳过体检分析、聊天、订阅、打卡、三个Tab全跳过。30秒可演示：评委扫瓶子 → 选"肾结石" → 红色警告。

## Approaches Considered

### Approach A: 极简单页（48小时原型）
- **Summary:** 只做"成分冲突检测"单页小程序，无Tab，无聊天
- **Effort:** S (2天)
- **Risk:** Low（技术简单），High（产品感太薄，创业赛道评审可能不认可）
- **Pros:** 最快可演示，最锐利的moment，30秒讲清楚
- **Cons:** 创业赛道看"产品商业能力"，单页可能显得产品不完整；无法展示商业模式闭环
- **Reuses:** 禁忌规则库、成分知识库

### Approach B: 平衡双Tab方案 [CHOSEN]
- **Summary:** Tab1 AI健康Agent（体检录入+AI对话+趋势图）+ Tab2 成分安全中心（冲突检测+成分查询+禁忌警告）
- **Effort:** M (10天人工 / CC辅助~2天)
- **Risk:** Medium（范围可控，但仍需17天内完成）
- **Pros:** 展示完整产品思路，冲突检测和AI对话双亮点，创业赛道评审三个维度都能覆盖
- **Cons:** 比单页复杂，需要更多开发时间
- **Reuses:** 禁忌规则库、成分知识库、DeepSeek对话模板

### Approach C: 完整三Tab（PRD原案）
- **Summary:** 三个Tab全做（Agent + 打卡提醒 + 选购内容）
- **Effort:** L (3周人工 / CC辅助~4天)
- **Risk:** High（17天内大概率做不完，质量会拉低）
- **Pros:** 最完整的产品形态，最好的商业故事
- **Cons:** 打卡和选购Tab对Demo贡献低，分散开发资源
- **Reuses:** 全部BMAD分析产出

## Recommended Approach

**Approach B: 平衡双Tab方案。**

理由：在17天限制下，砍掉打卡/选购Tab集中火力在两个真正有差异化的功能上。冲突检测制造"moment"（第二意见的建议），AI对话展示技术深度（创业赛道要求）。两者组合 = 完整的产品+商业故事。

**明确砍掉的范围：** Tab 3（选购内容 + 打卡提醒）推迟到黑客松后的Phase 2。Demo只展示Tab1（AI健康Agent）和Tab2（成分安全中心）。

### "成分形式级推荐"具体示例

**输入：** 用户体检显示血红蛋白偏低(110g/L)，有胃肠敏感史
**输出：**

> 建议补充铁剂。推荐**双甘氨酸亚铁(Ferrous Bisglycinate)**而非**硫酸亚铁(Ferrous Sulfate)**。
> - 双甘氨酸亚铁：吸收率~45%，胃肠刺激小，适合你的胃肠敏感史
> - 硫酸亚铁：吸收率~12%，常见副作用包括便秘、恶心
> - 建议剂量：25-30mg元素铁/天，空腹服用，搭配维C 100mg促进吸收
> - 服用时间：早餐前1小时 或 睡前空腹
> - 注意：与钙片间隔2小时以上（钙抑制铁吸收）

### 红/黄/绿严重度分级标准

| 等级 | 定义 | 示例 |
|------|------|------|
| 红色 | 硬性禁忌：有循证证据的直接危害 | 肾结石 + 高剂量维C（>1g/天）；华法林 + 维K |
| 黄色 | 交互/注意：剂量依赖或人群特异性 | 钙 + 铁（同服影响吸收，间隔2小时可解决）；孕妇 + 维A（剂量需控制） |
| 绿色 | 无已知冲突 | 鱼油 + 钙片；维D + 镁 |

## The Assignment

具体操作：把你搭档正在吃的保健品列表 + 他的病史，手动查禁忌关系（用Examine.com + PubMed），记录下：
1. 发现了几个真实冲突？
2. 最惊险的是哪个？（这就是Demo的开场故事）

## 市场风险深度分析

### 风险1: 合规雷区（大厂避开的原因）

**为什么蚂蚁阿福不做保健品推荐：**
- 保健品推荐一旦出问题（用户吃出问题），法律责任不明确
- "健康信息服务"和"医疗建议"的边界模糊
- 保健品行业本身灰色地带多，平台推荐可能被视为背书

**你的防御策略：**
- 禁忌规则硬编码（确定性逻辑，不依赖LLM）
- 每次输出附带免责声明（UI层强制）
- 严重异常强制就医（硬编码触发）
- 措辞严格：只用"辅助""参考""营养补充"
- 推荐算法与CPS佣金解耦

### 风险2: 留存地狱（4%的30天留存率）

**健康App留存失败的核心原因（来自PMC研究）：**
- 46%用户因"初始流程繁琐、隐藏费用、兴趣消退"停用
- 健康App依赖习惯养成，而习惯养成是缓慢且脆弱的过程
- 体检=低频（1年1次），如果只做体检分析，用完即走

**你的留存策略：**
- "成分冲突检测"是高频入口（每次买新保健品都可用）
- 裂变传播："你吃的这几样冲突吗？"分享卡片
- 后续可加：月度健康简报（推送回访理由）
- 长期：纵向数据积累形成迁移成本

### 风险3: 腾讯健康的碾压可能性

**腾讯健康2026年动态：**
- 已在微信内上线"健康管理助手"AI Agent
- 覆盖1万+医疗机构
- 正在开发微信AI Agent（灰度测试mid-2026）
