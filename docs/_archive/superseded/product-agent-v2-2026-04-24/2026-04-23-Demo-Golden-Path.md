# VitaMe 黑客松 Demo Golden Path 脚本

> 场景：WAIC 超级个体创业黑客松 2026 年 4 月底
> 时长目标：**2.5 分钟**（站台评委 demo），可裁剪到 1 分钟电梯版
> 演示核心：**P0 安全判断** + **Pivot 方向预埋（提醒钩子 + Memory 展示）**

---

## 1. 演示策略

### 1.1 选这个场景的三个理由

1. **最符合 P0 定位**："你给爸妈买的保健品，真的安全吗？"——Demo 必须是"帮爸妈看"而不是"给自己吃"
2. **Aha Moment 最强**：VC + 结石是教科书级别的交叉禁忌，非专业用户几乎不会知道。**"哦对不能吃"是产品价值瞬间具现的时刻**
3. **预埋 Pivot**：收尾的"设提醒"钩子展示了 P1/P2 的产品飞轮，让评委相信这不是一次性工具

### 1.2 人物设定

- **演示者（小王）**：30 岁产品经理，在台上扮演"周末回家的成年子女"
- **隐含人物（小王妈妈，58 岁）**：演示中提及的档案对象
  - 病史：双肾结石（2022 年体检发现 5mm）、血压正常高值
  - 正在吃：降压药（培哚普利）、钙片

### 1.3 开场一句话（抢评委注意力）

> "我前天回家发现我妈在抖音上买了一瓶 iHerb 的维生素 C 高剂量片——而我妈 2022 年做过肾结石手术。我们来看 VitaMe 怎么处理这件事。"

## 4. 评委可能的 5 个问题 + 答法

| Q | A |
|---|---|
| "这些规则谁审核？" | "团队有医学检测博士 + 临床护士。种子规则 32 条在 `docs/research/禁忌规则-v0-种子库-待医学审核-2026-04-23.md`，每条 P0 上线前都经双人审核。" |
| "LLM 会不会瞎编建议？" | "我们的设计是——**LLM 不做判断，只做翻译**。禁忌判断 100% 走硬编码规则引擎，LLM 只做 ① 商品名归一化 ② 机制说明的措辞个性化。出事可追溯到代码。" |
| "用户凭什么相信你不泄露数据？" | "Local-only 是默认模式（点开 HUD 看），端侧加密，LLM 实际看到的 token 可视化。我们不是在隐私上玩文字游戏——用户能亲眼看到。" |
| "和丁香医生 / 阿福的区别？" | "他们做'科普 + 导购'，我们做'判断 + 管理'。他们告诉你'鱼油有什么功效'，我们告诉你'你妈吃这瓶鱼油会反酸，因为她有溃疡病史'。" |
| "盈利模式？" | "短期：合规的替代补剂导流（非诱导）。长期：从补剂扩到处方药、饮食、运动的家庭健康管理订阅。今天 Demo 是切口，不是全部。" |

## 5. 需要预埋的假数据 / 工程最小集

| 项 | 实现方式 | 负责 |
|---|---|---|
| iHerb 链接解析 | 硬编码 3 条 demo 产品映射表（NOW VC、NOW 鱼油、Doctor's Best 镁） | 前端 |
| "妈妈" 档案 | 一键载入的 Demo 档案 JSON | 前端 |
| 规则库 | 本文件对应的 `seed-v0.json`（至少 10 条触发得到的 ✅/⚠️/❌） | 工程 |
| 提醒日程生成 | 硬编码 3 套示例日程（与 demo 档案匹配） | 前端 |
| Memory 计数器 | 前端状态管理（不需要后端） | 前端 |
| 隐私 HUD 展开 | 静态展示脱敏 token 样例 | 前端 |

**MVP 工程量估算**：前端 2 天 + 规则 JSON 1 天 + 文案/UI 1 天 = **4 天可上台**。

## 6. 失败预案

| 风险 | 应对 |
|---|---|
| 现场 Wi-Fi 不稳 / API 超时 | 全流程本地数据 + 本地规则引擎，不依赖网络 |
| 评委追问"真的能解析 iHerb 链接吗" | Demo 版诚实说"此版是 3 条示例映射，上线版将对接 iHerb / 京东解析 API" |
| 评委质疑"硬编码规则覆盖不够" | 主动展示规则库 JSON + 承诺 P0 扩到 50 条 + P1 扩到 200 条的计划 |
| 评委问"老人家不会用手机" | 设计定位从开始就是"**子女代用 + 生成家人档案**"，不是老人自己用 |

## Insights

- **Demo 必须"让评委看到原因"**：VC 禁忌的拦截不展开机制 = 又一个"黑盒 AI"。展开 "R-KD-01 + 草酸钙结晶" 这一句话，评委信任分瞬间翻倍。
- **Pivot 钩子放在倒数第 30 秒最佳**：太早放会盖过 P0 的安全 wedge；太晚放来不及让评委记住产品飞轮。T+01:35 是"证明完 Verify 价值 + 还没结束"的完美窗口。
- **隐私可视化是 demo 的"秘密武器"**：健康类产品的评委 90% 会问隐私问题。把"🔍 LLM 看到的"做成 UI 按钮而非 FAQ 条目，是所有其他健康 demo 没有的——评委会记住这一点。
