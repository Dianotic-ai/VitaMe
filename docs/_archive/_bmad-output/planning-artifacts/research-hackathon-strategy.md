## 二、MVP市场选择：国内优先

**结论：先做国内（中国市场），不犹豫。**

### 决策矩阵

| 维度 | 国内 | 国外 | 判定 |
|------|------|------|------|
| 黑客松适配 | WAIC是国内赛事，评委看国内场景 | 无关 | 国内 |
| 获客渠道 | 微信小程序 + 小红书种草 + 体检季投放 | 需Google Play/App Store，冷启动难 | 国内 |
| 技术红利 | 腾讯混元AI小程序计划：免费云资源 + 1亿token额度 | 无对应福利 | 国内 |
| 竞品空白 | AQ做健康问答但不做补剂推荐闭环，无直接竞品 | Labdoor/Examine.com已有成熟平台 | 国内 |
| 支付闭环 | 微信支付，CPS导购天然适配京东/天猫 | 需Stripe，跨境CPS复杂 | 国内 |
| 合规成本 | 不宣称疗效 + 三层安全架构，MVP可控 | FDA/FTC监管更严，需律师 | 国内 |
| 用户习惯 | 体检报告上传 → 微信场景天然 | 体检体系不同，需重新设计流程 | 国内 |
| 数据源 | 国标体检指标 + 国内补剂品牌数据库 | 需另建英文数据库 | 国内 |
| 市场规模 | 国内保健品市场持续增长 | 全球AI营养市场$17.72B by 2035 | 国外（Growth阶段再考虑） |

### 国内市场关键数据

- **AI个性化营养市场**：全球 USD 1.59B (2025) → USD 17.72B (2035)，CAGR 27.4%
- **国内竞品格局**：AQ（蚂蚁健康）用户增长60.1%，但无补剂推荐闭环
- **补剂分析平台**：Labdoor、Examine.com、ConsumerLab 均为英文平台，国内无对标产品
- **腾讯AI小程序计划2026**：免费云开发资源 + 1亿混元token配额

## 三、基于赛事评审的Demo策略

### 优化后的5分钟Demo结构

| 时间 | 内容 | 对应评审维度 |
|------|------|------------|
| 0:00-0:40 | 痛点故事：搭档的真实补剂踩坑经历 | 创意（真实需求） |
| 0:40-1:10 | 一句话方案 + "提前3年发现健康拐点" | 产品商业能力 |
| 1:10-3:00 | **Live Demo**：上传体检报告 → 趋势图 → AI推荐 → 购买链接 | 技术实现 |
| 3:00-4:00 | 商业模式：三线营收 + 腾讯AI小程序计划红利 + 冷启动100人策略 | 产品商业能力 |
| 4:00-4:40 | 技术架构：DeepSeek-V3 + RAG + 三层安全 + ¥74/月成本 | 技术实现 |
| 4:40-5:00 | 数据飞轮 + 扩展路线图 | 创意 + 商业 |

### 与PRD原始Demo方案对比

| 调整项 | 原PRD方案 | 赛事优化方案 | 原因 |
|--------|----------|-------------|------|
| 技术展示时长 | 2分钟 | 2.5分钟 | 创业赛道重技术实现 |
| 腾讯AI计划 | 未提及 | 重点强调 | 展示平台级资源支持 |
| 成本数据 | 未展示 | ¥74/月 | 证明商业可行性 |
| Demo完整度 | 趋势图+推荐 | 上传→分析→推荐→购买全闭环 | 创业赛道要求可运行产品 |

### Demo核心要求

- **必须跑通完整闭环**：上传体检报告 → AI分析 → 趋势图 → 补剂推荐 → 购买链接
- **不能断链**：任何一环掉链都会影响"产品商业能力"评分
- **数据要真实**：用真实体检报告数据，不要Mock数据

## 四、17天冲刺计划

距4月30日报名截止还有17天。

### 优先级排列

| 优先级 | 任务 | 时间 | 产出 |
|--------|------|------|------|
| **P0** | 报名注册 | Day 1 | 确保参赛资格 |
| **P0** | 体检报告上传 + OCR/解析 MVP | Day 1-5 | 核心数据入口 |
| **P0** | DeepSeek-V3对话 + 补剂推荐核心逻辑 | Day 3-8 | AI核心能力 |
| **P1** | 趋势图可视化（对比历史数据） | Day 6-10 | "Aha Moment"功能 |
| **P1** | CPS商品链接集成（京东联盟API） | Day 8-12 | 商业闭环证明 |
| **P2** | 微信小程序壳 + 基础UI | Day 10-14 | 可展示的产品形态 |
| **P2** | Demo录制 + PPT准备 | Day 14-17 | 参赛材料 |

### MVP砍掉的功能（Demo不需要）

以下功能属于Growth阶段，17天冲刺中不做：

- 打卡系统
- 社区功能
- 月度健康简报
- 游戏化元素（指标倒计时等）
- 组队打卡 + 裂变分享
- 年度健康报告

### 技术栈确认（MVP）

| 层 | 选型 | 原因 |
|----|------|------|
| 前端 | uni-app (Vue3) | 一套代码 → 微信小程序 + 后续App |
| 后端 | Python 3.11 + FastAPI | 快速开发，DeepSeek SDK原生支持 |
| AI | DeepSeek-V3 API（¥1/M tokens） | 成本极低，中文能力强 |
| 数据库 | SQLite | MVP够用，零运维 |
| 部署 | 腾讯云轻量应用服务器 | ¥74/月，适配微信生态 |
| Agent框架 | 暂不用Hermes Agent | MVP直接调DeepSeek API，降低复杂度 |
