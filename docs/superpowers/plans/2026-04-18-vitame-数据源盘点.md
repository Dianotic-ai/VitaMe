# VitaMe 数据源盘点

## 背景

VitaMe 核心场景:中国用户购买的保健品**既有海淘的国外产品**(iHerb 美国品牌、澳洲 Swisse、日本 DHC)**也有国产品**(汤臣倍健、善存、钙尔奇等),看不懂英文成分表 + 不知道能不能吃。

本文档盘点所有候选数据源,按**国别 + 用途**两维度展开,用于 P0 数据层设计决策的依据。

## 数据源评估维度

每个数据源按 4 维评分(各 ⭐~⭐⭐⭐⭐⭐):

- **Coverage**:覆盖目标用户的瓶子吗?
- **Authority**:权威性(政府机构 > 学术 > 商业 > 论坛)
- **Accessibility**:拿到的成本(公开 API > 公开下载 > 爬虫 > 付费 > 申请)
- **Freshness**:数据新鲜度(在维护 > 停更但可用 > 过期)

## P0 12 天冲刺最终选型

基于 coverage × effort × Demo 杀伤力三维取舍,**P0 确定使用**:

### 🟢 必做(8 个数据源)

| # | 数据源 | 用途 | 预估工程量 | 负责层 |
|---|---|---|---|---|
| 1 | NIH ODS Fact Sheets | 成分深度知识主源 | 1 人日 | L1 呈现层 |
| 2 | Linus Pauling Institute | 成分深度知识补强 | 0.5 人日 | L1 呈现层 |
| 3 | 中国营养学会 DRIs | 中国推荐剂量/上限 | 0.5 人日 | L1 呈现层 |
| 4 | PubChem / ChEBI | 化学形式标准化 | 0.3 人日 | L1 内部 |
| 5 | SUPP.AI | 补剂×药物交互 | 1 人日 | L2 判断层 |
| 6 | DSLD(Top 500 成分字典) | 成分名映射词典 | 0.5 人日 | L3 适配层 |
| 7 | 澳洲 TGA(Swisse/Blackmores) | 澳洲产品抽样 | 0.3 人日 | L3 适配层 |
| 8 | 日本機能性食品(DHC/FANCL) | 日本产品抽样 | 0.3 人日 | L3 适配层 |

### 🟡 P1 加分(2 个数据源,视时间投入)

| # | 数据源 | 用途 | 预估工程量 |
|---|---|---|---|
| 9 | DDInter | 药×药交互补强 | 0.5 人日 |

### 🔴 明确不做

- OpenFDA(过度,Demo 用不上)

- 京东/天猫(反爬严,后期 CPS 阶段再做)

## 3 层架构对齐

```
┌────────────────────────────────────────────────────────┐
│ L1 呈现层 (字典):NIH ODS + LPI + 中国 DRIs + PubChem │ → 50% 精力
├────────────────────────────────────────────────────────┤
│ L2 判断层 (报警器):SUPP.AI + DDInter + 50 硬编码禁忌 │ → 30% 精力
├────────────────────────────────────────────────────────┤
│ L3 适配层 (翻译器):DSLD 词典 + TGA + 機能性 + 蓝帽子 │ → 20% 精力
│    + Minimax 多模态 OCR                                │
└────────────────────────────────────────────────────────┘
```

## 法律与合规预警

| 数据源 | 使用方式 | 合规注意事项 |
|---|---|---|
| NIH/DSLD/PubChem | 离线烘焙 | 美国政府公共领域(public domain),无授权限制 |
| Linus Pauling | 离线烘焙 | 学术内容,注明来源 + 不得直接拷贝长段原文 |
| SUPP.AI | 离线烘焙 | Apache 2.0 / CC-BY 开源,注明 source_ref |
| DDInter | 离线烘焙 | CC-BY-NC-SA,非商业可用,注明来源 |
| TGA ARTG | 抽样爬 | 澳大利亚政府公开数据 |
| 日本機能性 | Excel 下载 | 日本消费者厅公开数据 |
| 蓝帽子 | 抽样手录 | 中国政府公开数据 |
| 中国 DRIs | 手录关键值 | 引用营养学会数值属于合理使用,但**全本书翻译会侵权** |

**关键**:所有数据源在产品输出时都要标注 `source_ref` 字段,Compliance 层在 UI 层展示"来源 X"按钮,这是黑客松评委审视"权威性"的证据链。
