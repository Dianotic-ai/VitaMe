## 1. 执行摘要

### 愿景

健康守护Agent是一个AI驱动的健康管理微信小程序，核心叙事：**"提前3年发现你的健康拐点"**。通过体检报告纵向趋势分析、保健品成分循证推荐（精确到化学形式和个人禁忌）、习惯打卡形成完整闭环，让亚健康人群对自身健康拥有掌控感。

### 差异化

市场上无产品同时串联"看懂体检→选对保健品→坚持吃"三个环节：
- **丁香医生**偏医疗科普，无保健品成分级解析
- **薄荷健康**聚焦饮食减脂，无体检分析
- **Examine.com**是英文学术数据库，无个性化推荐和中文化
- 无产品做多年体检报告纵向趋势分析（全是单次快照）

### 目标用户

**MVP聚焦人群**：25-35岁亚健康年轻白领。久坐办公，身体开始发信号（腰痛、眼干、肠胃不适），有改善意愿但不知从何入手。

**扩展人群**（后续版本）：中老年人、宝妈、健身人群。

### 商业模式

Phase 1 三线营收：单次体检深度报告 ¥9.9 + CPS导购佣金 5-15% + 会员订阅 ¥19.9/月。推荐算法与佣金解耦，透明披露合作关系。

## 4. 用户旅程

### UJ-1: 首次使用 — "我刚拿到体检报告，想看看有没有问题"

**角色**：小王，28岁程序员，刚做完年度体检

1. 小王在小红书看到"AI帮我发现体检隐患"的帖子，扫码进入小程序
2. 首页引导："上传体检报告或手动录入关键指标"
3. 小王手动录入5个关键指标（血糖、尿酸、ALT、LDL-C、血红蛋白）
4. AI 90秒内生成分析结果：
   - 各指标解读（正常/偏高/偏低+含义说明）
   - **同龄人对比**："你的尿酸420，高于同龄人平均值370"
   - 饮食和生活建议
5. 小王觉得有用，决定把去年的指标也录入
6. 录入第二年数据后，系统生成**纵向趋势图**：
   - "你的尿酸从2024年380升至2025年420，年增幅10.5%，按此趋势2027年将进入高风险区"
   - **Aha Moment命中**
7. AI推荐保健品方案：
   - "建议补充维C（500mg/天，缓释型），辅助降尿酸"
   - "推荐甘氨酸镁而非氧化镁（吸收率高10倍，且你有胃肠敏感史）"
   - 禁忌提示："注意：你有肾结石病史，维C每日不超过1g"
8. 小王点击推荐产品链接，跳转京东购买

**触发的功能需求**：FR-1, FR-2, FR-3, FR-4, FR-5, FR-6, FR-7, FR-8

### UJ-2: 成分冲突检测 — "我在吃的几种保健品会不会冲突？"

**角色**：小李，32岁设计师，已在吃3种保健品

1. 小李从朋友分享的小程序卡片进入"成分冲突检测"入口
2. 输入正在服用的保健品：钙片、鱼油、铁剂
3. AI分析结果：
   - ⚠️ "钙和铁不应同时服用，建议间隔2小时以上（钙会抑制铁的吸收）"
   - ✅ "鱼油和钙可以同服，无冲突"
   - 💡 "建议：铁剂空腹+维C服用，钙片随晚餐服用"
4. 小李觉得有用，分享给闺蜜群
5. 闺蜜们扫码使用，裂变完成

**触发的功能需求**：FR-5, FR-6, FR-9

### UJ-3: 深度咨询 — "我有具体症状，想问AI"

**角色**：小张，30岁金融从业者，最近腰痛+失眠

1. 小张进入AI对话Tab，描述症状："最近腰痛，晚上失眠，白天犯困"
2. AI追问关键信息：工作习惯、饮食、已有病史、正在服用的药物
3. 小张回答：久坐8小时、饮食不规律、无重大病史、无用药
4. AI生成综合评估：
   - 腰痛可能与久坐相关，建议每小时站立活动5分钟
   - 失眠可能与镁缺乏+压力有关
   - 推荐：甘氨酸镁（晚间300mg，助眠效果好）+ 维生素B6（辅助镁吸收）
   - ⚠️ 免责声明："以上建议仅供参考，如症状持续请就医"
5. 小张满意，决定开通会员获取更详细的个性化方案

**触发的功能需求**：FR-1, FR-3, FR-4, FR-6, FR-10

## 5. 领域要求（健康信息服务）

### DR-1: 法律合规定位
- 产品定位为**"健康信息服务"**，非"医疗服务"或"医疗器械"
- 小程序类目选择"生活服务-营养查询"，避免"医疗健康"类目审核
- 所有输出不得包含疾病诊断、治疗方案、处方药推荐

### DR-2: 免责声明
- 每次AI输出必须附带免责声明："本建议仅供参考，不构成医疗诊断，请遵医嘱"
- 首次使用时展示完整服务协议，用户需主动勾选同意
- UI层强制展示，非仅在条款中

### DR-3: 异常指标处理
- 检测到严重异常指标（如空腹血糖>7.0、ALT>120等）时，强制触发"请立即就医"提示
- 此规则硬编码于系统层，不依赖LLM输出

### DR-4: 数据隐私
- 体检数据属于《个人信息保护法》定义的敏感个人信息
- 需单独授权同意（与服务协议分离）
- 传输加密（TLS 1.3）+ 存储加密（AES-256）
- 用户可随时查看、导出、删除个人数据
- 数据最小化原则：只收集必要指标

### DR-5: 内容合规
- 避免使用"治疗""治愈""药效"等医疗用语
- 使用"辅助""参考""营养补充"等措辞
- 成分推荐标注证据等级，证据不足时标注"有限证据"
- 不得使用"循证"一词进行营销宣传（保健品领域高质量RCT稀少）

### 推荐技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端 | uni-app (Vue3) → 微信小程序 | 跨端复用，后续App |
| 后端 | Python 3.11 + FastAPI | MVP最快，生态成熟 |
| AI层 | DeepSeek-V3 API | 成本低(￥1/M tokens)、中文强、国内可达 |
| Agent框架 | FastAPI直调（MVP）→ Hermes Agent（正式版） | MVP速度优先 |
| 数据库 | SQLite（MVP）→ PostgreSQL（正式） | 零配置启动 |
| 部署 | 腾讯云轻量应用服务器 | ￥74/月 |
| OCR | 腾讯云/百度OCR API | P1阶段接入 |

### 安全架构（三层防护）

1. **硬编码规则层**：药物-成分禁忌对照表（~50条），确定性检查，无LLM幻觉风险
2. **Prompt约束层**：System Prompt限定角色边界和输出格式
3. **输出过滤层**：正则+关键词检测拦截诊断性表述（"你患有""建议用药"等）

### 月运营成本：~￥200-400
- DeepSeek API: ￥100-300（日均1000次对话）
- 腾讯云服务器: ￥74
- OCR: 免费额度内

## 8. 功能需求

### FR-1: AI对话健康评估
用户通过自然语言对话描述症状和健康问题，AI Agent基于用户已录入的体检数据、病史和当前症状，生成综合健康评估。每次对话输出包含：评估结论、建议措施、免责声明。AI每次回复延迟不超过10秒。

**关联**：UJ-1, UJ-3 → SC-1

### FR-2: 体检指标手动录入
用户可手动录入体检指标（指标名、数值、单位、日期）。系统支持至少20项常见指标（血糖、血脂、肝肾功能、血常规等）。录入界面提供指标名称模糊搜索，单次录入在5个操作步骤内完成。

**关联**：UJ-1 → SC-1

### FR-3: 纵向趋势分析
当用户录入2次及以上同一指标数据时，系统自动生成时间轴趋势图。趋势分析包含：方向性判断（上升/下降/稳定）、变化速率计算、参考范围标注、风险预警（连续2年同向偏移触发黄色预警，接近或超出参考范围触发红色预警）。

**关联**：UJ-1 → SC-2

### FR-4: 个性化保健品推荐
基于用户体检数据、症状描述和病史，AI推荐保健品方案。推荐精确到：成分名称、化学形式（如"甘氨酸镁"而非"镁"）、建议剂量范围、服用时间（饭前/饭后/空腹/晚间）、推荐理由（关联到哪个指标或症状）。每次推荐不超过5种成分。

**关联**：UJ-1, UJ-3 → SC-1, SC-6

### FR-5: 禁忌安全校验
系统维护禁忌规则库（~50条硬编码规则），覆盖常见病史×成分禁忌关系（如肾结石→高剂量维C，华法林→维K等）。每次推荐前自动校验用户病史与推荐成分的交叉关系。发现禁忌时，阻止该成分推荐并展示警告说明。禁忌校验为硬编码逻辑，不依赖LLM输出。

**关联**：UJ-1, UJ-2 → SC-3

### FR-6: 成分知识库查询
用户可查询成分信息，返回：中文名、英文名、化学形式列表及差异说明（吸收率、适用场景、价格级）、服用时间建议、常见禁忌。MVP覆盖Top 50高频成分。未收录成分返回"数据库暂未收录"提示，不进行猜测。

**关联**：UJ-1, UJ-2, UJ-3 → SC-6

### FR-7: 同龄人指标对比
首次录入单份体检报告时，系统基于用户年龄段和性别，展示关键指标与同龄人群平均值的对比。对比数据来源于公开统计数据（内置参考表）。此功能解决首次用户无纵向数据时的即时价值问题。

**关联**：UJ-1 → SC-1

### FR-8: 导购推荐链接
推荐保健品时，附带购买链接（京东联盟/iHerb等CPS链接）。链接处透明标注"含推荐合作链接"。推荐排序由AI成分匹配度驱动，不受佣金率影响。

**关联**：UJ-1 → SC-5

### FR-9: 成分冲突检测（独立入口）
提供独立功能入口"你吃的这几样冲突吗？"。用户输入2-5种正在服用的保健品/成分名称，系统检测相互作用和冲突关系，输出：冲突对及原因、建议服用间隔时间、优化服用时间表。支持微信小程序卡片分享检测结果。

**关联**：UJ-2 → SC-1

### FR-10: 病史与用药记录
用户可录入和管理个人病史（疾病名称、确诊时间）和当前用药列表（药物名称）。此数据用于禁忌校验和个性化推荐。数据加密存储，修改需二次确认。

**关联**：UJ-3 → SC-3

### FR-11: 免责声明展示
每次AI输出在结果底部展示固定免责声明文案。首次使用展示完整服务协议弹窗，用户需勾选同意后方可使用。检测到严重异常指标时，输出内容被替换为"请立即就医"强制提示（硬编码触发，不可被Prompt覆盖）。

**关联**：DR-2, DR-3 → SC-3

## 9. 非功能需求

### NFR-1: 性能
- AI对话响应时间不超过10秒（P95），通过后端日志监控
- 小程序首屏加载时间不超过3秒，通过微信开发者工具Performance面板测量
- 趋势图渲染时间不超过2秒（5年数据量）

### NFR-2: 安全
- 所有API通信使用HTTPS (TLS 1.3)
- 体检数据存储使用AES-256加密
- 用户认证使用微信OpenID，不存储用户密码
- AI输出经过三层安全过滤（硬编码规则→Prompt约束→关键词过滤）

### NFR-3: 隐私合规
- 符合《个人信息保护法》要求
- 敏感数据（体检指标、病史）需单独授权同意
- 用户可导出和删除个人数据，删除操作在24小时内完成
- 数据最小化：只收集功能必需的指标数据

### NFR-4: 可用性
- 小程序主包大小不超过2MB
- 核心功能（对话+录入+查看结果）在弱网环境(3G)下可用
- 关键操作（录入、提交）有加载状态反馈

### NFR-5: 可扩展性
- 成分知识库支持热更新，无需发版即可新增成分
- 禁忌规则库支持配置化管理，新增规则无需修改代码逻辑
- 后端API设计支持后续App和Web端复用

### NFR-6: 成本控制
- 月运营成本不超过￥500（MVP阶段）
- LLM调用采用缓存策略，相同指标组合的通用解读结果缓存复用
